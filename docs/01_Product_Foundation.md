# Constellate – AI 减脂领导力教练

## 一、产品定位

Constellate 是一个帮助用户提高自我管理能力、从而实现减脂目标的 AI 教练。

核心问题不是"吃多少"，而是：**为什么在明知目标的情况下，人仍然反复偏离？** 答案不是知识不足，而是自我领导力不足——在压力、疲劳、社交场景下维持一致性的能力。

Constellate 不教用户知识，而是训练用户在真实生活场景中的决策能力。它通过三种核心机制实现这一目标：

1. **结构化教练节奏**：每日晨间状态感知（Check-in）→ 日间关键时刻微干预 → 晚间结构化复盘，让行为改变成为持续运行的过程。
2. **LifeSign 预案系统**：帮助用户在清醒时为脆弱时刻预设应对方案（if-then 预案），通过预判性自我管理替代意志力消耗。
3. **签名体验（Signature Experience）**：在用户旅程 7 个关键节点生成个性化视觉干预图像，创造深层情感锚定（详见 4.1 节）。

产品的理论基础是 S-PDCA（State → Plan → Do → Check → Act）行为对齐循环，融合 EMA 情境采样、BCT 自我调节技术包和体验式教练干预（详见第二章）。这些理论框架内嵌于教练行为中，用户无需了解其存在。

## 二、理论基础

Constellate建立在四层行为改变理论闭环之上：

### 2.1 四层理论闭环

**1. Self-Monitoring（数据采集层）**
- 核心机制：Reactivity——仅记录就能改变行为
- 关键原则：一致性 > 频率（每周≥3天是效果阈值）
- 多维记录：时间戳 + 情境 + 情绪 + 生理状态

**2. EMA（情境解析层）**
- 生态瞬时评估：在真实环境中实时采样
- 多层次模型：情境 → 生理 → 情绪 → 行为
- 核心洞察：行为波动源于情境与状态的交互效应

**3. BCT（干预技术层）**
- 自我调节技术包：Self-monitoring + Goal setting + Feedback + Problem solving + Coping strategies
- 关键发现：应对计划（if-then预案）对长期维持效果显著

**4. 体验式教练干预（深化干预层）**
- 四类方法：未来自我对话、场景预演、隐喻协作、认知重构
- 核心原则：时机 > 形式；伦理优先（避免身体形象，聚焦行为能力）

### 2.2 从理论到S-PDCA

四层理论形成完整闭环，S-PDCA是其操作化：
- **State（调频优先）**：EMA状态采集 + BCT情绪调节
- **Plan（意图对齐）**：BCT目标设定 + 未来自我对话
- **Do（行为记录）**：Self-Monitoring + 多维事件记录
- **Check（模式识别）**：EMA模式识别 + 隐喻协作
- **Act（结构纠偏）**：BCT应对策略 + 场景预演 + 认知重构

核心原则：**State Before Strategy**——高压力/疲劳状态下的计划无效，优先恢复认知资源。

详细理论阐述见附录A。

## 三、目标人群与核心场景

Constellate 面向"明明知道该怎么做，却总是做不到"的人群——理性、上进、有目标的知识工作者或管理者。他们具备充足的信息储备（热量赤字、自律重要性、长期健康价值），但现实生活中的压力、疲劳、情绪波动和社交环境持续打断其节奏。

减脂作为入口场景，因饮食和身体状态每天都在发生、决策频繁、反馈明确，是训练"内在一致性"的理想领域。

Constellate 帮助用户在忙碌与压力之中，逐渐形成稳定的身份：**一个在复杂环境中仍能做出清晰选择的人**。服务对象为长期自我成长人群，而非短期减重需求。

## 四、系统逻辑

### 1. State（调频优先）

在高压力或疲劳状态下，系统优先触发微干预，使用户恢复可决策状态。

原则：**State Before Strategy**。

### 2. Plan（意图对齐）

将长期目标拆解为当日可执行边界。强调"身份一致性"，而非刚性数字。

### 3. Do（行为记录）

所有行为以"事件文件"形式记录，包含：时间戳、原始证据（图片/语音）、结构摘要、区间估算、情境标签与置信度。

### 4. Check（模式识别）

分析摄入区间趋势、情境变量、睡眠影响与行为波动。

关键问题：**行为波动发生前的触发条件是什么？**

当识别到重复模式时，Coach 提议创建 LifeSign 预案（if-then 应对计划），帮助用户为高风险情境预设应对方案。

### 5. Act（结构纠偏）

系统调整策略、建议环境改变、更新观测维度。**阻抗被视为信息，而非错误。**

LifeSign 预案的成功执行触发正向强化：累计成功达到里程碑时，系统生成签名体验图像确认身份演化。

## 4.1 核心教练机制

以下三大机制是产品定位（第一章）所述核心能力的设计定义。操作级实现细节见 Coach Agent prompt。

### Onboarding

首次使用时引导用户完成自我画像建立。目标是让 Coach 从第一天起具备针对性干预能力。

采集内容：用户身份与减脂动机、自我管理卡点（高频失控场景）、��风险时段与场景、每日 Check-in 自选追踪维度（体重/体脂/睡眠/饮食计划）、通知偏好。

Onboarding 以启程仪式结束——系统生成第一张签名体验图像（future_self），标志教练关系正式建立。随后 Coach 协作生成用户的第一个 LifeSign 预案。

### 每日教练节奏

三个结构化触点贯穿每一天，训练用户的自我觉察、决策暂停和自我反馈能力：

| 触点 | 训练的自我管理能力 | 设计意图 |
|------|-----------------|---------|
| **晨间 Check-in** | 自我觉察 | 感知今日状态，Coach 据此决定当天策略（State Before Strategy） |
| **日间微干预** | 决策点暂停 | 在高风险时刻主动触达，引用 LifeSign 预案，打断冲动决策 |
| **晚间复盘** | 自我反馈 | 结构化回顾当天选择，Coach 提供 L1-L3 层级反馈���见第六章） |

Check-in 基础维度（能量/情绪/压力）为所有用户默认启用，可选维度由用户在 Onboarding 中勾选。

### LifeSign 预案系统

LifeSign 是结构化的 if-then 应对预案，帮助用户在清醒时为脆弱时刻预设应对方案��

预案生命周期：协作生�� → 存储 → 情境匹配 → 提醒触发 → 用户执行 → 效果追踪 → 迭代优化。

生成时机：Onboarding 后首次教练对话、模式识别后（Check 阶段）、用户主动提及挑战、失控事件后复盘。

追踪机制：每次匹配情境记录激活次数，成功执行记录成功次数。累计成功 3 次触发里程碑签名体验图像。

### 签名体验（Signature Experience）

签名体验是按用户旅程阶段部署的视觉干预，在关键节点创造情感锚定。

| 旅程阶段 | 触发条件 | 图像类型 | 情感风格 |
|---------|---------|---------|---------|
| 启程仪式（Day 1） | Onboarding 完成 | future_self | 暖色/人情味 |
| 首个 LifeSign（Day 1-3） | 首个预案协作完成 | scene_rehearsal | 冷色/精确 |
| 首个模式洞察（Day 7-14） | Coach 识别到有意义的行为模式 | reframe_contrast / metaphor_mirror | 根据信息正负 |
| LifeSign 成功里程碑 | 某预案累计成功 3 次 | identity_evolution | 暖色/人情味 |
| 高风险场景预判 | 用户提及即将面对的挑战 | scene_rehearsal | 冷色/精确 |
| Chapter 完成（Day 21） | Chapter 结束 | identity_evolution | 暖色/人情味 |
| 低谷支持 | 连续多天状态差或用户表达挫败 | metaphor_mirror / reframe_contrast | 根据信息正负 |

图像风格遵循双轴系统：正面信息（鼓励、成长）→ 暖色/人情味风格；负面/中性信息（分析、风险）→ 冷色/精确风格。理论依据：负面信��用精确风格降低防御心理，正面信息用人情味增强情感连��。

## 五、数据结构

Constellate 采用行为文件系统，而非传统数据库：

- 每日为目录，每次行为为带时间戳的事件文件
- 原始证据永久保存，结构层可演化
- LifeSign 预案独立存储于 `/user/coping_plans/`，包含触发条件、应对行为、激活与成功计数
- 用户偏好（通知频率、Check-in 维度）存储于 `/user/profile/`

派生层生成趋势、模式识别与情境-行为关联。Coach Agent 拥有指标治理能力，可新增或退役观测维度。

## 六、Guardrail：伦理边界与反馈机制

### 6.1 内容伦理边界

**禁止内容**：
- 理想化身材图像或身体对比
- 任何聚焦于体重/体型/外观的干预
- 身体形象焦虑诱导

**聚焦内容**：
- 行为能力与情境掌控
- 身份演化与价值观具象化
- 应对策略与模式识别

### 6.2 反馈层级系统（L1-L5）

Coach根据用户状态、数据积累程度、行为模式清晰度，选择恰当的反馈层级：

**L1 - 确认式反馈**（Do阶段，即时）
- 触发：用户完成行为记录
- 内容："记下了。午餐摄入在目标区间内。"
- 情感：平静确认，不评判

**L2 - 评估式反馈**（Do阶段，当日）
- 触发：单日数据完整
- 内容："今天三餐节奏很稳，摄入在1600-1800区间。"
- 情感：数据先行，结构化

**L3 - 趋势式反馈**（Check阶段，每周）
- 触发：周度复盘
- 内容："这周工作日摄入在目标区间，周五晚较高。"
- 情感：模式呈现，不预设问题

**L4 - 模式式反馈**（Check阶段，≥3周数据）
- 触发：显著模式出现
- 内容："连续三周周五晚摄入偏高，都在应酬场景后。"
- 情感：识别触发，提供洞察

**L5 - 身份式反馈**（Act阶段，Chapter复盘）
- 触发：Chapter完成或显著转变
- 内容："连续两周工作日早餐结构一致——这是节奏稳定的信号。"
- 情感：身份强化，长期视角

### 6.3 应对计划机制

**if-then预案生成**（对应BCT 1.4 Action Planning）：

触发条件：
- Check阶段识别到重复风险模式
- 用户主动表达担忧（"明天有聚餐"）
- 场景预演干预中

结构：
```
IF [触发情境] → THEN [应对行为]
示例："IF 下午3点感到疲劳 → THEN 先喝一杯水，走5分钟，再决定是否需要零食"
```

关键：
- 具体情境线索（时间、地点、生理状态）
- 可执行的微行为（而非抽象意志）
- 用户共创（Coach提议，用户确认或修改）

### 6.4 用户控制与透明度

**干预前确认**：
- Coach在生成体验式干预前询问意愿
- 用户可随时关闭体验式干预功能

**干预后反馈**：
- 每次干预后收集反馈（有帮助/不舒服/跳过）
- 反馈结果影响后续干预选择

**退出机制**：
- 用户可删除Map页任意教练洞察卡片
- 删除记录不影响Coach对用户的理解

**生成内容标识**：
- 所有AI生成内容明确标识为"可能性探索，而非预测"

## 七、Agent 架构与运作方式

用户只面对一个"教练"。教练负责倾听、观察上传的照片、理解当天发生的事情，并决定何时聊天、何时记录、何时提醒。

系统不对每句话过度反应。仅在与目标相关时整理信息，并结合近期数据做出判断。系统记住每天发生的事情，但不以复杂数字施压，而是关注趋势与模式。所有分析在后台完成，用户只面对一个冷静、稳定的教练。

核心：**让改变成为持续运行的过程，而非靠意志力硬撑的冲刺。**

## 八、扩展方向

减脂是首个落地场景。同一架构（S-PDCA + 教练节奏 + LifeSign）可扩展至运动训练、睡眠优化、情绪管理等需要长期行为一致性的领域。

## 附录A：理论基础详细阐述

### A.1 Self-Monitoring：持续记录是改变基础

**Reactivity（反应性）定义**：
仅仅因为观察和记录行为，行为本身就会改变，无论记录准确性如何。实质是提高意识（awareness），打断自动化行为模式，激活有意识决策。

**关键发现**：
- **一致性 > 频率**：每周≥3天记录是效果阈值，宁可每天记录少量，不要间断多日
- **多维记录 > 单维记录**：时间戳 + 情境 + 情绪 + 生理状态
- **即时记录 > 回忆记录**：当天晚上回忆早餐，准确性已显著下降

**Self-Monitoring的真正目的**：识别模式，而非监督行为。阻抗本身就是信息。

### A.2 EMA：情境变量是行为的真实驱动

**生态瞬时评估（Ecological Momentary Assessment）**在用户自然环境中实时采样行为与体验，捕捉情境对行为的即时影响。

**多层次关联模型**：
```
情境变量（环境、社交、时间）
  ↓
生理状态（睡眠、疲劳、饥饿）
  ↓
情绪状态（压力、沮丧、无聊）
  ↓
行为输出（饮食选择、摄入量）
```

**三种采样方法**：
1. **信号触发（signal-contingent）**：随机信号采集背景状态
2. **事件触发（event-contingent）**：用户主动记录关键行为
3. **间隔触发（interval-contingent）**：固定时间点采样

**S-PDCA整合**：
- State使用signal-contingent
- Do使用event-contingent
- Check识别模式

### A.3 BCT：有效干预的技术组件

**行为改变技术（Behavior Change Technique）**研究揭示：单一技术无一致效果，**自我调节技术包**是减重干预中唯一一致的预测因子。

**自我调节技术包五要素**：
1. Self-monitoring（自我监测）
2. Goal setting（目标设定）
3. Feedback（行为反馈）
4. Problem solving（问题解决）
5. Coping strategies（应对策略）

**关键发现**：
- 应对计划（if-then预案）对长期维持的效果 > 单纯行动计划
- 个性化反馈 > 通用反馈（减重差异2.1 kg）
- 即时反馈 > 延迟反馈（用户黏性+40%）

**BCT与S-PDCA映射**：
- State：情绪调节技术（BCT 11.1-11.3）
- Plan：目标设定（BCT 1.1-1.3）+ 行动计划（BCT 1.4）
- Do：Self-monitoring（BCT 2.3）
- Check：反馈机制（BCT 2.2）+ 问题解决（BCT 1.2）
- Act：应对计划（BCT 1.2）+ 认知重构（BCT 13.2）

### A.4 体验式教练干预：深化行为改变的关键时刻

教练干预的核心不在于"表达形式"，而在于**时机、内容、情感**。

**四类干预方法**：

**1. 未来自我对话（Future Self Continuity）**
- 理论基础：Hershfield的未来自我连续性研究
- 时机：目标设定、动机低落、长期复盘
- 内容：身份一致性优先于外观变化
- 情感：希望感、掌控感、连续性

**2. 场景预演（Scenario Rehearsal）**
- 理论基础：Mental Contrasting (MCII/WOOP) + Episodic Future Thinking
- 时机：识别到高风险模式后、用户主动请求、预防性干预
- 内容：情境线索 + 应对行为 + if-then链接
- 情感：平静、掌控感、准备就绪

**3. 隐喻协作（Metaphor Collaboration）**
- 理论基础：Lakoff & Johnson的概念隐喻理论
- 时机：用户使用隐喻语言、状态调频、模式识别后
- 内容：识别 → 镜射 → 协作精化（增加资源，不改变源域）
- 情感：被理解、赋能感、可能性

**4. 认知重构（Cognitive Reframing）**
- 理论基础：CBT核心机制 + Dual-Coding Theory
- 时机：认知扭曲、陷入自责、复盘时
- 内容：框架对比（失败 → 信息）、比例感呈现
- 情感：同理、信息感、可通过性

**伦理约束**：
- ❌ 禁止：理想化身材、身体对比、任何聚焦于体重/体型的内容
- ✅ 聚焦：行为能力、情境掌控、价值观具象化

**部署与频率**：见 4.1 节"签名体验"的旅程阶段部署表。
