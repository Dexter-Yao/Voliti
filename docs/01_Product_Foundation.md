# Voliti – AI 减脂行为教练

## 一、产品定位

Voliti 是一个 AI 减脂行为教练。它提供饮食指导、运动建议、方案制定与每日跟进——这些是任何教练都应该做的事。

Voliti 真正解决的问题不是"不知道该怎么做"，而是**"知道该怎么做但反复做不到"**。用户研究（[07_User_Research.md](07_User_Research.md)）显示：62% 的目标用户每周至少经历一次"明知不该但还是做了"，55% 的反复尝试者一次失控就演变为 2-3 天的崩溃。所有人都会失控——差别在于一次失控是一个点，还是一个三天的坑。

**Coach 的核心价值：让你摔倒后更快站起来，站起来后更少摔倒。** 具体来说，它关注三个绝大多数工具和方法忽略的时刻：

1. **失控之前**——识别用户的高风险模式，在用户还清醒的时候帮他预设应对方案（LifeSign 预案系统）
2. **失控之后**——在用户破功的第二天帮他恢复，把"破功"从一个自我审判降级为一个数据点，防止单次失控螺旋化（State Before Strategy）
3. **跨时间的模式**——发现用户自己看不到的行为规律，比如"每次加班后都会失控"（时间感知系统）

一句话总结：**让减脂从"反反复复"变成"偶尔绕路但一直在走"。**

产品的理论基础是 S-PDCA（State → Plan → Do → Check → Act）行为对齐循环，融合 EMA 情境采样、BCT 自我调节技术包和体验式教练干预（详见第二章）。这些理论框架内嵌于教练行为中，用户无需了解其存在。

## 二、理论基础

Voliti建立在四层行为改变理论闭环之上：

### 2.1 四层理论闭环

**1. Self-Monitoring（数据采集层）**
- 核心机制：Reactivity——仅记录就能改变行为
- 关键原则：一致性 > 频率（每周≥3天是效果阈值）
- 多维记录：时间戳 + 情境 + 情绪 + 生理状态

**2. EMA（情境解析层）**
- 生态瞬时评估：在真实环境中实时采样
- 多层次模型：情境 → 生理 → 情绪 → 行为
- 核心洞察：行为波动源于情境与状态的交互效应

**3. BCT（干预技术层）**
- 自我调节技术包：Self-monitoring + Goal setting + Feedback + Problem solving + Coping strategies
- 关键发现：应对计划（if-then预案）对长期维持效果显著

**4. 体验式教练干预（深化干预层）**
- 四类方法：未来自我对话、场景预演、隐喻协作、认知重构
- 核心原则：时机 > 形式；伦理优先（避免身体形象，聚焦行为能力）

### 2.2 从理论到S-PDCA

四层理论形成完整闭环，S-PDCA是其操作化：
- **State（调频优先）**：EMA状态采集 + BCT情绪调节
- **Plan（意图对齐）**：BCT目标设定 + 未来自我对话
- **Do（行为记录）**：Self-Monitoring + 多维事件记录
- **Check（模式识别）**：EMA模式识别 + 隐喻协作
- **Act（结构纠偏）**：BCT应对策略 + 场景预演 + 认知重构

核心原则：**State Before Strategy**——高压力/疲劳状态下的计划无效，优先恢复认知资源。

详细理论阐述见附录A。

## 三、目标人群与核心场景

Voliti 面向"明明知道该怎么做，却总是做不到"的人群——理性、上进、有目标的知识工作者或管理者。他们具备充足的信息储备（热量赤字、自律重要性、长期健康价值），但现实生活中的压力、疲劳、情绪波动和社交环境持续打断其节奏。详细用户画像与数据支撑见 [`07_User_Research.md`](07_User_Research.md)。

减脂作为入口场景，因饮食和身体状态每天都在发生、决策频繁、反馈明确，是训练"内在一致性"的理想领域。

Voliti 帮助用户在忙碌与压力之中，逐渐形成稳定的身份：**一个在复杂环境中仍能做出清晰选择的人**。服务对象为长期自我成长人群，而非短期减重需求。

## 四、系统逻辑

### 1. State（调频优先）

在高压力或疲劳状态下，系统优先触发微干预，使用户恢复可决策状态。

原则：**State Before Strategy**。

### 2. Plan（意图对齐）

将长期目标拆解为当日可执行边界。强调"身份一致性"，而非刚性数字。

### 3. Do（行为记录）

所有行为以"事件文件"形式记录，包含：时间戳、原始证据（图片/语音）、结构摘要、区间估算、情境标签与置信度。

### 4. Check（模式识别）

分析摄入区间趋势、情境变量、睡眠影响与行为波动。Coach 同时持有即时视角（单次对话）和长期旅程视角（跨周分析），识别重复触发链和行为衰减信号。

关键问题：**行为波动发生前的触发条件是什么？**

当识别到重复模式时，Coach 提议创建 LifeSign 预案（if-then 应对计划），帮助用户为高风险情境预设应对方案。前瞻标记中即将到来的事件与历史模式关联时，Coach 进行预防性准备。

### 5. Act（结构纠偏）

系统调整策略、建议环境改变、更新观测维度。**阻抗被视为信息，而非错误。**

LifeSign 预案的成功执行触发正向强化：累计成功达到里程碑时，Coach 生成 Witness Card 见证用户的身份演化。

## 4.1 核心教练机制

以下四大机制是产品定位（第一章）所述核心能力的设计定义。操作级实现细节见 Coach Agent prompt。

### Onboarding（渐进式披露）

首次使用时通过分层引导建立用户画像并创造首次教练体验。核心原则：**最小必要信息先行，后续自然补采**。

**必答层**（所有用户必须完成）：
1. 用户称呼 + 未来自我愿景（身份式表达，非数字目标）——写入 `profile/context.md`，成为独立顶层概念
2. 当前与愿景的距离感知（State 评估）
3. 结果目标与时间约束——创建 `/goal/current.json`，成为独立概念
4. 启程仪式——Coach 生成首张 Witness Card，见证教练关系正式建立

**扩展层**（必答层完成后自然延伸）：
5. Plan Scaffold——Coach 辅助拆解 Goal，创建首个 Chapter（Goal 之下的时间阶段，包含 title、milestone、3 个 Process Goal）
6. 高风险场景认领——从常见失控场景中选出"对我最危险的"，为首批 LifeSign 预案提供输入
7. 近期前瞻事件采集——了解用户未来 2-4 周的已知安排，写入前瞻标记作为时间轴冷启动

**后续补采**（自然对话中完成）：
- 更多失控场景与对应 LifeSign
- Check-in 追踪维度偏好
- 通知偏好

**后续补采**：未在 Onboarding 中采集的信息由 Coach 在自然对话中适时收集。Coach 持续追踪已知信息的空白，在给出建议前自检上下文是否充分——不足则保守回应并主动补采。

### 典型每日教练节奏

三个结构化触点贯穿每一天，训练用户的自我觉察、决策暂停和自我反馈能力：

| 触点 | 用户价值 | 设计意图 |
|------|-----------------|---------|
| **晨间 Check-in** | 自我觉察 | 感知今日状态，参考前瞻标记调整当天策略（State Before Strategy） |
| **日间微干预** | 决策点暂停 | 在高风险时刻主动触达，引用 LifeSign 预案，打断冲动决策 |
| **晚间复盘** | 自我反馈 | 结构化回顾当天选择，Coach 提供 L1-L3 层级反馈���见第六章） |

Check-in 基础维度（能量/情绪/压力）为所有用户默认启用，可选维度由用户在 Onboarding 中勾选。Check-in 在当日首次打开 App 时自动触发（`[daily_checkin]` 系统触发器），Coach 通过 A2UI fan_out 呈现轻量状态收集界面，结合 Coach Briefing 中的近期回顾为对话提供连续性。

### LifeSign 预案系统

LifeSign 是结构化的 if-then 应对预案，帮助用户在清醒时为脆弱时刻预设应对方案。

**核心原则：匹配优先于创建。** 当用户描述的情境与已有 LifeSign 匹配时，Coach 必须引用已有预案而非创建新的。当前瞻标记中的即将到来事件匹配已有 LifeSign 时，Coach 主动关联并提前引入预案讨论。

预案生命周期：协作创建 → 存储 → 情境匹配 → 引用触发 → 用户执行 → 效果追踪 → **动态迭代**。

生成时机：Onboarding 场景认领（首批预案）、模式识别后（Check 阶段）、用户主动提及挑战、失控事件后复盘、前瞻标记触发的预防性准备。

追踪机制：每次匹配情境记录激活次数，成功执行记录成功次数。累计成功达到里程碑时触发 Witness Card。

**动态更新**：LifeSign 的 action 不是固定的——如果用户反馈某个预案持续无效，Coach 协作修改 action；如果用户发现更有效的应对方式，更新预案。

### 时间感知（Temporal Awareness）

Coach 维护一条双向时间轴，由三类条目组成：行为记录（回顾性，已有事件文件）、生活事件（回顾性，用户提及的经历）、前瞻标记（前瞻性，用户提及的未来安排）。时间是索引轴，因果推理由 Coach 自行完成。

**即时视角**：Coach 在每次对话中参考前瞻标记，使状态感知和微干预具备上下文。

**长期旅程视角**：后台定期分析用户时间轴，识别重复行为模式和即将到来的高风险事件，将分析摘要注入 Coach 对话上下文。干预基于个人行为信号检测，而非固定时间窗口。

**设计原则**：陪伴优先，工具其次。信息采集边界由 Coach 根据用户开放程度自行判断。主动干预必须渐进、有据、在用户授权范围内。

### 教练干预体系

Coach 的干预手段是**对话 + A2UI 结构化交互**，不使用图片。四类干预方法（未来自我对话、场景预演、隐喻协作、认知重构）均通过 Coach 文字引导配合 A2UI 组件（slider 做状态评估、select 做 if-then 选择、text_input 做自由反思）实现，充分发挥大语言模型的对话智能。

```
SESSION 内（聊天窗口）
├── 对话式教练 ─── Coach 文字对话（核心干预手段）
│   ├── 状态评估 ─── A2UI（slider, select）
│   ├── 场景预演 ─── A2UI + 文字引导
│   ├── 认知重构 ─── A2UI + 文字引导
│   ├── 隐喻协作 ─── A2UI + 文字引导
│   └── 未来自我 ─── A2UI + 文字引导
│
│ ↓ 当用户达成里程碑时
│
└── Witness Card ─── FanOutPanel（品牌框架 + 图片 + 文字）
    → 用户收下 → 存入收藏
    → 用户拒绝 → Coach 尊重，继续对话

SESSION 间（主动触达，P2）
├── 前瞻提醒 ─── Forward Marker 触发
├── Check-in ─── 时间节奏触发
├── 模式洞察 ─── JourneyAnalysis 触发
└── 回看引导 ─── 低动力期触发（引导用户回看 Witness Cards）
```

干预成功本身可以成为见证触发：Coach 帮用户预演了一次商务晚宴应对方案（干预），用户事后告诉 Coach"我选了气泡水"（成功），Coach 为这个时刻生成 Witness Card（见证）。

### 见证系统（Witness Card）

Witness Card 是用户旅程中里程碑时刻的个性化纪念卡片，不是干预工具。它的价值是**让用户被看见**，并为未来提供力量。

**定位**：见证（witnessing），不是奖励（rewarding）。Coach 生成 Witness Card 的语气是"我记录了你的这个时刻"，而不是"给你一个奖品"。这一区分基于自我决定理论（SDT）：信息性反馈支持内在动机，控制性奖励侵蚀内在动机。

**卡片结构**：统一品牌框架 + AI 生成的场景图片（无文字叠加）+ Coach 撰写的个性化叙事文字。图片和文字在框架内分区呈现，文字独立可读。

**触发逻辑**：由 Coach 自主判断，不硬编码指标。Coach 可参考的信号包括：
- 显性成就：用户主动告知的成功（"我昨晚扛住了"）
- 隐性成就：Coach 通过 ledger 数据发现的用户自己未察觉的进步
- 旅程节点：Chapter 完成、LifeSign 累计成功里程碑、行为模式正向转变
- 启程仪式：Onboarding 完成时的首张 Witness Card

隐性成就的见证尤为重要——用户没意识到自己做到了什么，Coach 主动为他生成 Witness Card，这个"被看见"的体验是教练关系的核心价值。

**呈送方式**：通过 FanOutPanel（A2UI interrupt）呈现，用户可选择收下或拒绝。发送 Witness Card 的过程本身就是一种教练引导，是对用户行为和经历的纪念性记录。

**图片风格**：Coach 在统一视觉体系内选择色温方向（暖色调用于成长/温柔时刻，冷色调用于突破/力量时刻），品牌框架、纸张质感、排版结构保持一致。

**收藏与回看**：已接受的 Witness Card 存入用户收藏。在低动力期，Coach 可引导用户回看历史 Witness Cards，利用怀旧驱动的自我连续感重新激发目标追求行为。

**频率约束**：每 Chapter（约 21 天）约 3-5 张，不在每次对话中生成。里程碑的稀缺性是其情感重量的来源。

## 五、数据结构

Voliti 采用行为文件系统，而非传统数据库：

- 用户身份与设置存储于 `/profile/context.md`，作为独立顶层概念
- 结果目标存储于 `/goal/current.json`（当前目标）与 `/goal/archive/{id}.json`（历史目标），成为独立概念
- Chapter 是 Goal 之下的时间阶段，存储于 `/chapter/current.json`，包含 title、milestone、3 个 Process Goal。Process Goal 是 Chapter 内的行为目标，与 dashboardConfig 的 support_metrics 一一对应
- LifeSign 预案存储于 `/lifesigns.md`，简化 markdown 格式，最多 6 条，每条不超过 100 字（触发场景 → 应对策略）
- 每日为目录，每次行为为带时间戳的事件文件；原始证据永久保存，结构层可演化
- 前瞻标记存储于 `/timeline-calendar.md`，markdown 按日期分组，支持 Coach 直接编辑与日终清理

派生层（`/derived/`）生成 briefing 与候选信号。Coach Agent 拥有指标治理能力，可新增或退役观测维度。日终 Pipeline 自动封存当天会话、生成日摘要（`/day_summary/`）并更新 briefing。

## 六、Guardrail：伦理边界与反馈机制

### 6.1 三层边界体系

**Layer 1 — 可自由回答**：通用卡路里估算、通用宏量营养素建议、通用饮食方案讨论（间歇性断食原理、生酮机制等）、行为模式识别与反馈、饮食计划协助。

**Layer 2 — 保守回答 + 转介**：疾病相关饮食建议（甲减/糖尿病/GI 等），提供一般参考但明确建议遵医嘱或咨询注册营养师。Coach 遵循「上下文置信度原则」——对用户情况不够了解时保守回应并主动补采信息。

**Layer 3 — 绝对红线**：心理危机（自伤/自杀意念/严重饮食障碍）→ 共情 + 转介专业帮助；药物/补剂建议 → 永远转介；用户施压下验证明确有害行为为自我关爱 → 区分共情与认同，保持温暖但不后退。

### 6.1.1 内容伦理边界

**禁止内容**：理想化身材图像或身体对比、任何聚焦于体重/体型/外观的干预、身体形象焦虑诱导。

**聚焦内容**：行为能力与情境掌控、身份演化与价值观具象化、应对策略与模式识别。

### 6.2 反馈层级系统（L1-L5）

Coach根据用户状态、数据积累程度、行为模式清晰度，选择恰当的反馈层级：

**L1 - 确认式反馈**（Do阶段，即时）
- 触发：用户完成行为记录
- 内容："记下了。午餐摄入在目标区间内。"
- 情感：平静确认，不评判

**L2 - 评估式反馈**（Do阶段，当日）
- 触发：单日数据完整
- 内容："今天三餐节奏很稳，摄入在1600-1800区间。"
- 情感：数据先行，结构化

**L3 - 趋势式反馈**（Check阶段，每周）
- 触发：周度复盘
- 内容："这周工作日摄入在目标区间，周五晚较高。"
- 情感：模式呈现，不预设问题

**L4 - 模式式反馈**（Check阶段，≥3周数据）
- 触发：显著模式出现
- 内容："连续三周周五晚摄入偏高，都在应酬场景后。"
- 情感：识别触发，提供洞察

**L5 - 身份式反馈**（Act阶段，Chapter复盘）
- 触发：Chapter完成或显著转变
- 内容："连续两周工作日早餐结构一致——这是节奏稳定的信号。"
- 情感：身份强化，长期视角

### 6.3 应对计划机制

**if-then预案生成**（对应BCT 1.4 Action Planning）：

触发条件：
- Check阶段识别到重复风险模式
- 用户主动表达担忧（"明天有聚餐"）
- 场景预演干预中

结构：
```
IF [触发情境] → THEN [应对行为]
示例："IF 下午3点感到疲劳 → THEN 先喝一杯水，走5分钟，再决定是否需要零食"
```

关键：
- 具体情境线索（时间、地点、生理状态）
- 可执行的微行为（而非抽象意志）
- 用户共创（Coach提议，用户确认或修改）

### 6.4 用户控制与透明度

**Witness Card 控制**：
- Witness Card 通过 FanOutPanel 呈送，用户可选择收下或拒绝
- 拒绝不影响 Coach 对用户的理解，Coach 尊重用户选择
- 已收下的 Witness Card 可随时删除

**干预控制**：
- 文字 + A2UI 干预在自然对话中进行，用户可随时选择不参与（skip/reject）
- Coach 根据用户反馈调整干预频率和方式

**生成内容标识**：
- 所有 AI 生成内容明确标识为"可能性探索，而非预测"

## 七、Agent 架构与运作方式

用户只面对一个"教练"。教练负责倾听、观察上传的照片、理解当天发生的事情，并决定何时聊天、何时记录、何时提醒。

系统不对每句话过度反应。仅在与目标相关时整理信息，并结合近期数据做出判断。系统记住每天发生的事情，但不以复杂数字施压，而是关注趋势与模式。所有分析在后台完成，用户只面对一个冷静、稳定的教练。

核心：**让改变成为持续运行的过程，而非靠意志力硬撑的冲刺。**

## 八、扩展方向

减脂是首个落地场景。同一架构（S-PDCA + 教练节奏 + LifeSign）可扩展至运动训练、睡眠优化、情绪管理等需要长期行为一致性的领域。

## 附录A：理论基础详细阐述

### A.1 Self-Monitoring：持续记录是改变基础

**Reactivity（反应性）定义**：
仅仅因为观察和记录行为，行为本身就会改变，无论记录准确性如何。实质是提高意识（awareness），打断自动化行为模式，激活有意识决策。

**关键发现**：
- **一致性 > 频率**：每周≥3天记录是效果阈值，宁可每天记录少量，不要间断多日
- **多维记录 > 单维记录**：时间戳 + 情境 + 情绪 + 生理状态
- **即时记录 > 回忆记录**：当天晚上回忆早餐，准确性已显著下降

**Self-Monitoring的真正目的**：识别模式，而非监督行为。阻抗本身就是信息。

### A.2 EMA：情境变量是行为的真实驱动

**生态瞬时评估（Ecological Momentary Assessment）**在用户自然环境中实时采样行为与体验，捕捉情境对行为的即时影响。

**多层次关联模型**：
```
情境变量（环境、社交、时间）
  ↓
生理状态（睡眠、疲劳、饥饿）
  ↓
情绪状态（压力、沮丧、无聊）
  ↓
行为输出（饮食选择、摄入量）
```

**三种采样方法**：
1. **信号触发（signal-contingent）**：随机信号采集背景状态
2. **事件触发（event-contingent）**：用户主动记录关键行为
3. **间隔触发（interval-contingent）**：固定时间点采样

**S-PDCA整合**：
- State使用signal-contingent
- Do使用event-contingent
- Check识别模式

### A.3 BCT：有效干预的技术组件

**行为改变技术（Behavior Change Technique）**研究揭示：单一技术无一致效果，**自我调节技术包**是减重干预中唯一一致的预测因子。

**自我调节技术包五要素**：
1. Self-monitoring（自我监测）
2. Goal setting（目标设定）
3. Feedback（行为反馈）
4. Problem solving（问题解决）
5. Coping strategies（应对策略）

**关键发现**：
- 应对计划（if-then预案）对长期维持的效果 > 单纯行动计划
- 个性化反馈 > 通用反馈（减重差异2.1 kg）
- 即时反馈 > 延迟反馈（用户黏性+40%）

**BCT与S-PDCA映射**：
- State：情绪调节技术（BCT 11.1-11.3）
- Plan：目标设定（BCT 1.1-1.3）+ 行动计划（BCT 1.4）
- Do：Self-monitoring（BCT 2.3）
- Check：反馈机制（BCT 2.2）+ 问题解决（BCT 1.2）
- Act：应对计划（BCT 1.2）+ 认知重构（BCT 13.2）

### A.4 体验式教练干预：深化行为改变的关键时刻

教练干预的核心不在于"表达形式"，而在于**时机、内容、情感**。

**四类干预方法**：

**1. 未来自我对话（Future Self Continuity）**
- 理论基础：Hershfield的未来自我连续性研究
- 时机：目标设定、动机低落、长期复盘
- 内容：身份一致性优先于外观变化
- 情感：希望感、掌控感、连续性

**2. 场景预演（Scenario Rehearsal）**
- 理论基础：Mental Contrasting (MCII/WOOP) + Episodic Future Thinking
- 时机：识别到高风险模式后、用户主动请求、预防性干预
- 内容：情境线索 + 应对行为 + if-then链接
- 情感：平静、掌控感、准备就绪

**3. 隐喻协作（Metaphor Collaboration）**
- 理论基础：Lakoff & Johnson的概念隐喻理论
- 时机：用户使用隐喻语言、状态调频、模式识别后
- 内容：识别 → 镜射 → 协作精化（增加资源，不改变源域）
- 情感：被理解、赋能感、可能性

**4. 认知重构（Cognitive Reframing）**
- 理论基础：CBT核心机制 + Dual-Coding Theory
- 时机：认知扭曲、陷入自责、复盘时
- 内容：框架对比（失败 → 信息）、比例感呈现
- 情感：同理、信息感、可通过性

**伦理约束**：
- ❌ 禁止：理想化身材、身体对比、任何聚焦于体重/体型的内容
- ✅ 聚焦：行为能力、情境掌控、价值观具象化

**干预手段**：四类方法均通过 Coach 文字对话 + A2UI 结构化交互实现（详见 4.1 节"教练干预体系"）。干预成功后可触发 Witness Card 见证用户的成就（详见 4.1 节"见证系统"）。
