# Voliti 设计理念 – Starpath Protocol

> 相关文档：精确的 tokens、色值、组件规格见项目根目录 [`DESIGN.md`](/DESIGN.md)。
> 产品定位与理论基础见 [`01_Product_Foundation.md`](01_Product_Foundation.md)。

## 一、设计哲学

Voliti 的视觉与交互建立在两个原则之上：

1. **Identity First**：界面强化"成为那种人"的连续性，而非数字评判。
2. **Map Over Metrics**：用户走在路径上，而不是被仪表盘监控。

这两个原则决定了所有视觉决策的方向：不追求激励感，不使用游戏化元素，不用色彩评判行为好坏。界面的目标是营造一个让身份缓慢演化的空间。

## 二、视觉语言

核心风格：**Obsidian & Parchment + Copper（Refined Industrial）**

**为什么选择这个方向：**

Voliti 的用户是理性的知识工作者。竞品（Noom、Gentler Streak、Headspace）都在"让你感觉好"的赛道上竞争，用柔和蓝、圆角卡片、温柔语气。Voliti 的定位是"策略顾问"，不是"温柔的朋友"。

Obsidian & Parchment 的双色调传递的是工具感和克制。Copper 信号色不是装饰，而是 Coach 在关键时刻的视觉签名（预案触发、北极星指标、趋势活跃点）。

**排版三层的设计意图：**

| 层 | 语义 | 为什么 |
|----|------|--------|
| 叙事层（Serif） | Coach 的声音 | 楷体（LXGW WenKai）有书写温度，区别于通用宋体，让 Coach 的话语有人文质感 |
| 界面层（Sans） | 用户的声音 | 几何无衬线（DM Sans）干净中性，不抢 Coach 的存在感 |
| 信号层（Mono） | 系统的声音 | 等宽字体（JetBrains Mono）承担数据标签、时间戳、状态指示，是界面的结构骨架 |

**视觉原则：**

- 高对比、大留白、线性分割、少容器化
- 零圆角容器（仅 pill 按钮 999px）
- 无奖励动画、无激励色彩、无游戏化元素

## 三、信息层级结构

Voliti 的信息分四个语义层，通过 2 个页签承载：

1. **Coach Layer（交互层）**：用户的主要停留空间。Coach 通过对话和扇出组件动态呈现信息。
2. **Horizon Layer（身份层）**：长期趋势、身份演化、一致性指数。
3. **Path Layer（行为层）**：行为节点的时间线、风险区标记。
4. **Journal Layer（叙事层）**：结构性反思与数据追踪。

**页签映射：**

| 页签 | 承载的信息层 | 设计意图 |
|------|------------|---------|
| COACH | Coach Layer（主）+ 按需扇出其余三层 | 对话是唯一录入入口。Coach 决定何时、以何种形式呈现其余信息层。 |
| MIRROR | Horizon + Path + Journal | 纯展示，不录入。用户在这里"回看自己"，而不是"操作系统"。 |

**MIRROR 的设计逻辑：**

MIRROR 从上到下按信息权重递减排列：身份（Chapter）→ 核心指标（北极星）→ 支撑指标 → 行动力（LifeSign/预案）→ 行为历史（事件流）。用户打开 MIRROR，第一眼看到的是"我是谁"和"我的核心数字"，而不是流水账。

**Coach 扇出机制：**

扇出半 UI 是 Coach Layer 的核心表达工具。Coach 根据用户状态和交互需要，在对话区域上方以半屏或全屏面板呈现结构化信息。扇出组件目录包括：

- **信息呈现类**（Coach → 用户）：趋势卡片、LifeSign 预案卡片
- **数据采集类**（用户 → Coach）：饮食确认、状态签到、称重记录
- **微干预类**（Coach 主动触发）：Protocol Prompt Card，在检测到高压力或疲劳信号时触发

## 四、交互原则

- 冷静、精准、非评判式语言
- 优先呈现结构信息，再提供叙事
- 干预频率克制
- 不使用鼓励式动画或奖励机制

系统像一位沉稳的策略顾问，而非健身教练。

## 五、节奏设计

四个节奏点构成用户的日常循环：

| 节奏 | Coach 交互方式 | 设计意图 |
|------|---------------|---------|
| 行为后即时轻反馈 | 对话回复 + 可扇出饮食确认卡片或状态确认 | 利用 Reactivity 效应，记录本身改变行为 |
| 每日晚间结构复盘 | 扇出当日总结组件（半屏或全屏） | 强化"检查"环节，建立反思习惯 |
| 每周趋势总结 | 扇出趋势可视化 + MIRROR 页签 Coach 注解 | 从日常噪音中提炼信号 |
| 每月身份回顾 | 扇出身份演化回顾 | 强化长期身份叙事，不是短期成绩单 |

强调连续性，而非即时成就。
