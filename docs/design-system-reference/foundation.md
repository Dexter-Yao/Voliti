# Foundation — Starpath Protocol

## 产品定位

Voliti 是一套基于教练协议的长期行为对齐系统。减脂是第一落地场景，但产品核心不是管理卡路里，而是训练用户在**状态（State）—意图（Intention）—行为（Action）**之间建立一致性。

它填补效率工具与心理疗愈之间的真空：不追求外在生产力优化，也不进行病理式治疗，而是解决"内在动力阻滞"。

核心问题不是"吃多少"，而是：**为什么在明知目标的情况下，人仍然反复偏离？**

## 目标用户 JTBD

**目标用户**：知识工作者和管理者，理性、上进、有目标，信息储备充足，但现实生活中的压力、疲劳、情绪波动和社交环境持续打断节奏。

**核心 JTBD**：不需要更多信息或更强意志力。需要一个能够在混乱中维持内在结构的系统——让他们在复杂环境中仍能做出清晰的选择。

**不适合的用户**：寻求快速减重方案的人、需要即时奖励反馈的人、希望被激励式语言驱动的人。

## 目标情感基调

用户与 Voliti 交互时应感受到：

- **冷静权威**：像与一位沉稳的策略顾问对话，而非健身教练或激励演说家
- **被理解**：系统记住上下文，不要求用户重复解释，行为波动被视为信息而非错误
- **掌控感**：用户是决策者，Coach 是分析工具，不是裁判
- **连续性**：每一次交互都是长期路径的一部分，没有"开始"和"失败"的二元结构

## 类别信号（建立合法性的约定）

Voliti 属于健康/行为追踪类应用，以下类别约定需要保留以建立合法性：

- **信息层次清晰**：数据可查询、可追溯，用户不需要猜测系统在做什么
- **可操作的反馈**：每次 Coach 回应都有具体的行为线索，不是泛泛而谈
- **一致的语言风格**：整个产品使用同一套词汇体系，不在不同页面切换风格

## 差异化架构（刻意偏离类别约定之处）

以下是 Voliti 相对于同类应用的刻意偏离，每条偏离均服务于具体的情感目标：

| 偏离点 | 类别约定 | Voliti 选择 | 服务的情感目标 |
|--------|---------|-----------------|--------------|
| 色彩语言 | 激励橙绿或医疗蓝 | 黑曜石 + 羊皮纸 | 冷静权威、档案感，而非兴奋感或临床感 |
| 进度反馈 | 进度条、完成百分比 | 路径叙事、模式识别 | 连续性身份感，而非离散成就感 |
| 失败处理 | 红色标记、警告状态 | 所有数据视为信息 | 非评判式氛围，阻抗被视为分析材料 |
| 奖励机制 | 徽章、连续打卡、动画庆祝 | 无显式奖励 | 内在动机优先，不建立外部依赖 |
| 数据呈现 | 精确数字（如 1847 kcal） | 区间估算（如 1600-1800 kcal） | 减少数字焦虑，聚焦趋势而非计数 |
| 界面节奏 | 实时更新、高频推送 | 克制的触发条件 | 减少监控感，强化"教练在后台工作"的感知 |
