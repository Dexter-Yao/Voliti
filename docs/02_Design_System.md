# Voliti 设计系统 – Starpath Protocol

## 一、设计哲学

Voliti 的视觉与交互建立在两个原则之上：

1. **Identity First**：界面强化"成为那种人"的连续性，而非数字评判。
2. **Map Over Metrics**：用户走在路径上，而不是被仪表盘监控。

## 二、视觉语言

核心风格：**Obsidian & Parchment**

- 背景：温暖灰白（羊皮纸感）
- 主色：深黑曜石
- 点缀色：克制红（风险标记）
- 对齐状态：低饱和冷色

强调：

- 高对比
- 大留白
- 线性分割
- 少容器化

避免：

- 过度渐变
- 激励式色彩
- 医疗蓝

Coach 扇出面板遵循同一视觉体系，附加约束：

- 面板背景使用半透明羊皮纸色，与对话区域形成轻层次感
- 面板边界以线性分割为主，不使用阴影或浮层效果
- AI 生成图片须遵守 Obsidian & Parchment 色调，避免高饱和或激励式画面

## 三、信息层级结构

Voliti 界面分四层，通过三个页签承载：

1. **Coach Layer（交互层）**：用户的主要停留空间。Coach 通过对话和扇出半 UI 动态呈现下列三层信息。
2. **Horizon Layer（身份层）**：展示长期趋势、一致性指数与身份演化。
3. **Path Layer（行为层）**：以时间轴与轨迹线呈现行为节点与风险区。
4. **Journal Layer（叙事层）**：每日或周期性复盘。强调结构性反思，而非长篇文章。

页签映射：

| 页签 | 承载的信息层 | 职责 |
|------|------------|------|
| Coach | Coach Layer（主）+ 按需扇出其余三层 | 对话、扇出半 UI、Agent 主导的交互 |
| Map | Coach卡片归档 | Chapter元数据 + 体验式干预卡片列表 |
| Journal | Journal Layer | 账本数据、结构性复盘 |

**Map页设计哲学**：
- 不再承载Horizon Layer与Path Layer的直接呈现
- 转为Coach在关键时刻生成的教练洞察归档
- 设计原则："Map Over Metrics"——用户在地图上，不是在数轴上
- 每个卡片包含：AI生成图像 + Coach文案 + 时间戳 + 干预类型
- 用户可删除卡片、全屏查看、提供反馈
- 详细线框规范见`/docs/04_UI_Specification.md`第五节

Coach 扇出半 UI 是 Coach Layer 的核心机制——Coach 根据用户状态和交互需要，从预定义组件目录中选择组件，在对话区域上方以半屏或全屏面板呈现。用户随时可返回对话。

Map 和 Journal 页签支持 Coach 轻量注解：Coach 可在这两个页签的特定位置写入文本标注，辅助用户理解数据中的模式与趋势。

## 四、关键组件

### 4.1 基础视觉元素

跨页签通用的视觉构件：

- **Timeline Node**：细线 + 小圆点，用于 Journal 页签的行为节点标记。
- **Obsidian Pill**：主交互按钮，全圆角黑色胶囊。用于所有页签。

### 4.2 Coach 扇出组件目录

Coach 通过扇出半 UI 向用户呈现或采集信息。所有扇出组件是 Coach 的表达工具，由 Coach 根据用户状态和交互需要动态选择。

**信息呈现类**（Coach → 用户）：

- 趋势卡片：可视化摄入区间、体重变化等趋势数据
- 模式识别卡片：展示 Coach 发现的行为模式与触发条件
- 应对计划卡片：呈现if-then预案供用户确认

**数据采集类**（用户 → Coach）：

- 饮食确认卡片：用户语音描述后，Coach 解析并扇出结构化确认界面
- 状态签到卡片：用于 State 阶段快速采集当前身体/情绪状态
- 称重记录卡片：简洁的数值输入与趋势对比

**微干预类**（Coach 主动触发）：

- Protocol Prompt Card：用于 State 微干预与决策引导。风格简洁、克制、无情绪化语言。Coach 在检测到高压力或疲劳信号时触发。

所有扇出组件遵循统一的视觉规范（见第二节）和交互原则（见第五节）。

## 五、交互原则

- 冷静、精准、非评判式语言
- 优先呈现结构信息，再提供叙事
- 干预频率克制
- 不使用鼓励式动画或奖励机制

系统像一位沉稳的策略顾问，而非健身教练。

## 六、节奏设计

- 行为后即时轻反馈
- 每日晚间结构复盘
- 每周趋势总结
- 每月身份回顾

各节奏点与 Coach 交互机制的对应：

| 节奏 | Coach 交互方式 |
|------|---------------|
| 行为后即时轻反馈 | 对话回复 + 可扇出饮食确认卡片或状态确认 |
| 每日晚间结构复盘 | 扇出当日总结组件（半屏或全屏） |
| 每周趋势总结 | 扇出趋势可视化 + Map 页签 Coach 注解 |
| 每月身份回顾 | 扇出身份演化回顾（可含 AI 生成图片） |

强调连续性，而非即时成就。

Voliti 的视觉与交互目标不是"激励"，而是营造一个让身份缓慢演化的空间。它让用户感觉自己不是被管理，而是在书写一条长期路径。
