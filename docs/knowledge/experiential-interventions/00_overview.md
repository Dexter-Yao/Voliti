# 体验式干预手段总览

本文件是四份分册（01-04）的引导图，目的是让 Coach 看见四种手段之间的关系，而不是提供"何时用哪个"的决策表。具体是否使用、如何使用、如何与用户对话，由 Coach 基于用户实际情境自行判断。本文档只给判断所需的维度语言，不给判断结果。

## 1. 四手法的差异轴

| 维度 | future-self-dialogue | scenario-rehearsal | metaphor-collaboration | cognitive-reframing |
|---|---|---|---|---|
| 时机倾向 | Plan 阶段 / 长复盘 | Plan → Act 连接 / 高风险事件前 | State / Check / 用户自发表达时 | Check / Act / 失控后复盘 |
| 情感基调 | 希望、连续性、身份确认 | 平静、掌控、准备就绪 | 被理解、赋能、可能性 | 同理、信息感、可通过性 |
| 用户自发度要求 | 无（Coach 可主动开场） | 无（Coach 可主动开场） | **用户先使用隐喻**，Coach 才介入 | 无（Coach 可主动开场） |
| 核心理论根基 | Future Self Continuity（Hershfield）/ 可能自我（Markus & Nurius）/ 立志与日新（儒家） | MCII/WOOP（Oettingen）/ Implementation Intentions（Gollwitzer）/ 庙算豫立 | 概念隐喻（Lakoff & Johnson）/ Clean Language（Grove）/ 比兴取象 | CBT（Beck）/ REBT（Ellis）/ 观心格物 |

## 2. 与 S-PDCA 的映射（提示，非约束）

| 手法 | 主要接入阶段 |
|---|---|
| future-self-dialogue | Plan |
| scenario-rehearsal | Plan → Act |
| metaphor-collaboration | State / Check |
| cognitive-reframing | Check / Act |

实际使用不限于此。Coach 判断阶段与状态契合度，而非按表机械匹配。

## 3. 跨手法共享的边界

以下条目适用于任何体验式干预，Coach 在触发前自检：

- **State Before Strategy** —— 用户若处于失调状态（高压、疲惫、情绪突击），先协同调频，任何手法后置。
- **失败容忍 > 打卡奖励** —— 干预的目的是让用户恢复前行能力，不是强化单次成功。
- **单次会话不堆叠** —— 一次会话内不连续调用两种手法；给上一种手法的效果足够呼吸空间。
- **硬停条件** —— 心理健康危机信号、用户明确拒绝、会话末端疲惫态，任何手法一律不触发。

## 4. 关于东方传统的使用口径

四份分册同时列出西方主流理论与东方传统视角，后者是**文化参照**，不是"与西方理论对等的替代体系"。

- 当用户呈现出文化倾向、语言风格或哲学取向时，Coach 可在措辞与框架选择上优先采用东方路径的锚点。
- 当用户偏好未明时，默认采用西方主流轴的措辞（证据基础更充分、对话形态更标准化）。
- 东方传统中的概念（如"本来面目""庙算""比兴""观心"）在分册中已标注"借鉴性对应，非严格对等"。Coach 使用时不向用户呈现这类元话语，只借用其文化表达形式。
