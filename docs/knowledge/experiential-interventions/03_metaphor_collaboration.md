# 隐喻协作（Metaphor Collaboration）

## 1. 理论基础

### 1.1 西方主流轴

隐喻不仅是修辞工具，而是认知与行动的基础架构。西方隐喻疗法的理论脉络由三个并行的经典框架组成：

**概念隐喻理论（Conceptual Metaphor Theory，CMT）**

George Lakoff 与 Mark Johnson 在1980年发表的《Metaphors We Live By》（中文译《我们赖以生存的隐喻》）奠定了现代隐喻理论的基石。CMT 的核心观点是：隐喻不仅存在于诗歌和艺术语言中，更深层地结构了人类日常思维。人们通过将抽象的、难以直接体验的概念（如"心态""成长"）映射到具体的、物理维度的领域（如"容器""方向""移动"），从而理解复杂的心理现象（Lakoff & Johnson, 1980）。例如，当用户说"我被困住了"或"我正在向上爬"时，这些表达背后的隐喻结构（DIFFICULTY IS A BARRIER / PROGRESS IS UPWARD MOVEMENT）已经形成了他对自身处境的认知框架。这一观点的深远意义在于：*改变隐喻，可能改变思维方式，进而改变行为选择*。

**Clean Language 与象征隐喻疗法（Clean Language & Symbolic Modelling）**

David Grove 在1980年代发展的 Clean Language 方法论提供了隐喻协作的实操框架。Grove 发现当人们用隐喻描述创伤或困境时，可以在"安全的隐喻空间"内进行心理探索，而无需直接重现痛苦的具体细节（Grove, 1989）。Clean Language 的原则是：治疗师（或教练）不向来访者提供自己的隐喻或框架，而是反复使用来访者自己的语言，通过一系列精心设计的中立问题（如"那个 X 像什么？""当你说 Y 时，什么会发生？"）引导来访者深化对自身隐喻的探索，最终自发地发现新的可能性。

James Lawley 与 Penny Tompkins 基于 Grove 的工作，系统化地发展了象征隐喻疗法（Symbolic Modelling），发表于著作《Metaphors in Mind: Transformation Through Symbolic Modelling》（2000）。象征隐喻疗法的关键洞察是：人的行为与困境往往由一组相互关联的内部隐喻组成（称为"隐喻格局"metaphor landscape）。通过有序地引导来访者在隐喻空间中进行自我对话与重新整合，可以实现认知、情感与行为层面的同步转变（Lawley & Tompkins, 2000, pp. 45-78）。

**ACT 中的隐喻应用（Acceptance and Commitment Therapy）**

Steven Hayes 及其同事在接纳与承诺疗法（ACT）中大量使用隐喻，作为帮助来访者实现认知解离（cognitive defusion）的手段。经典隐喻如"棋盘隐喻"（你的思想是棋子，但你是棋盘本身）旨在帮助来访者改变与困扰性思想的关系，而非试图改变思想本身（Hayes, Masuda, & De Mey, in press）。隐喻在 ACT 中的功能是"转化体验方式"：通过在体验层与符号层之间建立新的联系，来访者可以接纳痛苦思想的存在，同时继续朝着个人价值观行动。这一框架特别适用于行为改变情境，因其强调*与困难共存而非消除困难*。

**心理学实证支持**

关于隐喻在心理治疗中的有效性，公开学术文献显示：隐喻可以增强记忆留存（相比字面描述），支持"观察性距离"（observational distancing），进而增加心理改变的概率（Wagener, 2018）。在认知行为疗法（CBT）和精神动力疗法中，治疗师都普遍采用"外化隐喻"（externalizing metaphors）来帮助来访者将困扰性症状视为"外部对象"而非"自我的一部分"，从而打开改变空间（Metaphors in Professional Counseling, 2018）。

### 1.2 东方传统视角

东方传统的隐喻思维与西方 CMT 的关键区别在于：西方强调隐喻的*认知映射*，东方强调隐喻的*悟道功能*与*本体论地位*。

**《诗经》的"比兴"手法**

《诗经》采用"赋、比、兴"三种表现手法，其中"兴"最接近现代意义上的隐喻启发。朱熹在《诗集传》中界定："兴者，先言他物以引起所咏之辞也"（朱熹，《诗集传》）。这里的关键在于"引起"——通过提及一个具体的自然或生活现象，唤起读者的联想与情感，从而对所要表述的主题形成深层认同。例如《诗经·周南·关雎》开篇"关关雎鸠，在河之洲"，通过雎鸠鸣叫的自然场景，引起对爱情专一、伉俪和谐的深层感受。这与西方 CMT 的映射不同，东方"兴"的目的不是将抽象概念映射到具体领域，而是通过*共鸣*让接受者自发地体悟。朱熹的创新在于为每首诗每一章标注"赋也、比也、兴也"的组合，揭示了这些手法的有机整合。

**庄子的寓言与"无言之言"**

庄子自述其作文秘诀为"寓言十九，重言十七，卮言日出"，即通过虚拟人物、故事讲述与历史人物之口来表达思想（《庄子·寓言篇》记载）。庄子寓言的特点是"虚实相生"：通过讲述一个虚拟故事（如《逍遥游》中的"鲲鹏"），传达对人生自由本质的哲学思考。与西方隐喻强调"来源域→目标域"的线性映射不同，庄子寓言呈现的是一种"让心与物相通"的过程。寓言中的人物、动物或事件本身并无标准答案，而是邀请读者在不同观点间"齐物"——站在更高维度认识多个视角的等价性。陈鼓应在其《庄子内篇释义》中强调，庄子内篇呈现的是"开放的心灵与审美的心境"，这种心境正是隐喻启发的前提条件。

**禅宗公案与"不立文字"的启发**

禅宗公案（如《碧岩录》《无门关》中的案例）采用悖论性、超逻辑的隐喻或故事来打破来访者的常规思维，引发突然顿悟（悟）。圆悟克勤大师编纂的《碧岩录》包含百则公案，每则由"垂示""公案""着语""颂古""评唱"五层组成，层层递进引导参禅者进行内在探索（圆悟克勤，《碧岩录》，宋代）。禅宗公案与 Clean Language 的相似之处在于都采用"非直接指导"的方式：禅师不提供答案，而是通过一个隐喻性的问题或故事启发参禅者自发地顿悟。无门慧开的《无门关》强调"无门"——所有通往真理的"门"本都不存在，智慧来自对思维本身的超越。这一观点与 ACT 中"接纳而非改变"的逻辑有异曲同工之处。

**中医的"取象比类"思维**

中医学在《黄帝内经》中建立了一套基于"取象比类"的诊疗体系。张其成在其研究中阐明：中医通过观察自然界中五行（木火土金水）的属性，与人体五脏的功能形成对应，进而通过这套隐喻系统进行诊断与治疗（张其成，《中医象思维研究》）。例如，"肝象木而曲直，心象火而炎上，脾象土而安静"（《黄帝内经·素问·五脏生成论》）——这不是简单的比喻，而是一套本体论的隐喻体系，其中"象"（image/symbol）成为了医学诊疗的基本工具。这种思维方式的深刻性在于：通过隐喻的往来运动（从自然象征到人体现象），医者获得了一种整体性认识，超越了解剖学式的部分论述。

**借鉴性对应与现代整合**

需要说明的是，上述东方传统视角与西方 CMT 的直接对应在学术文献中证据有限。东方强调隐喻的*悟道与整体感受*，西方强调隐喻的*认知映射与机制*，两者的融合更多是*借鉴性而非严格对等*。然而，当代跨文化心理学实践（如正念认知疗法 MBCT、与诗歌疗法的整合实验）已开始探索东西融合的隐喻协作方式。心理咨询领域的整合实践表明，对那些有传统文化背景或哲学倾向的来访者，采用"取象比类"或禅宗公案式的提问往往比直接的西方隐喻映射更为有效。

## 2. 标准使用方法

### 2.1 西方 CMT + Clean Language 路径

**第一步：识别与镜射**

当用户在对话中自发使用了隐喻语言（如"我像一只漏气的气球""感到被困在围墙里"），Coach 首先需要精确识别这个隐喻的核心意象。此时 Coach 的任务不是解释隐喻，而是将其以最保真的方式镜射回用户。

示例：用户说"我最近总是被工作淹没，感觉不到喘气的机会"。Coach 的镜射可以是："你说被工作淹没，喘不了气。"（保留用户的确切措辞，不添加、不解释、不评价）。

这一步的关键是**保持中立与精确**。任何对用户隐喻的解释、评判或替代，都会破坏隐喻空间的纯净，进而减弱协作的深度。

**第二步：引导深化**

通过一系列中立的问题，邀请用户进一步探索隐喻内部的结构与含义。Grove 将这些问题称为"开放的清洁提问"（clean questions），典型的如：

- "那个 [用户的隐喻词汇] 像什么？"
- "当 [隐喻语句] 时，会发生什么？"
- "那个 [隐喻] 来自哪里？"（追溯隐喻的起源或触发点）
- "如果 [隐喻] 变化了，会怎样？"

这些问题的威力在于它们**强迫用户进行自我对话**，而非被动接受教练的诠释。在这个过程中，隐喻的边界开始明晰，隐喻中的矛盾或新可能性逐渐浮现。

示例延续：Coach 可以问"那个'被淹没'的感受像什么具体的场景？""在那个场景里，你的身体是什么样的？"通过这些问题，用户可能会发现"被淹没"其实意指"节奏失控"或"被动应对"，而非真的"无能为力"。

**第三步：协作精化**

当用户的隐喻达到足够的清晰度与自我理解后，Coach 可以邀请用户想象隐喻的演变。这是象征隐喻疗法的核心：不是改变隐喻中代表"自我"的角色（这样会导致虚假的改变），而是邀请隐喻内部的资源或可能性浮现。

示例：Coach 可以问"如果那道围墙有一扇门呢？""你怎么知道那扇门在哪里？"或更直接的："你需要穿过那道墙，还是需要跨出来站在墙之外？"这些问题不是在教导，而是在帮助用户发现自己已有但未被激活的可能性。

### 2.2 东方传统路径（禅宗/庄子风格）

**第一步：镜射与止语**

识别用户的隐喻后，Coach 的第一个动作是沉默的镜射——将隐喻表述回去，但接以一个"开放的静默"，而非立即的追问。

示例：用户说"我像一棵被风吹弯的草"。Coach 的回应可能仅是："一棵被风吹弯的草。"然后等待。这种"非填充式"的等待本身就是一种启发——它邀请用户进行内在观照。

**第二步：悖论性或超逻辑的提问**

采用禅宗公案式的提问，打破常规的逻辑预期，引发更深层的思考。这些提问通常不是"为什么"或"怎样"，而是一个看似无答案的悖论。

示例延续：Coach 可能问"那棵草知道自己在弯吗？"或"风停了，草会记得它被弯过吗？"这些提问没有正确答案，但强迫用户跳出"困境解决"的思维模式，进入"整体观照"的视角。

**第三步：取象比类的启蒙**

在必要时，Coach 可以引入自然或生活中的平行隐喻，帮助用户在"象"的层面进行思考，而非逻辑层面。这类似于中医的类推方式。

示例：如果用户被"失控"的困扰困住，Coach 可能引入"四季更替""潮汐涨退"等自然象征，帮助用户理解"失控"不是"异常"而是"自然循环"的一部分。

这一路径的优势在于：对于具有东方文化背景或更多整体化思维倾向的用户，它避免了西方路径中可能出现的"过度分析化"，而是在象征与感受层面进行工作。

## 3. 适用场景与条件

### 3.1 触发条件（何时启动隐喻协作）

隐喻协作不是"总是启动"的干预，而是**被动触发的**。以下条件表明可以考虑启动隐喻协作：

1. **用户自发使用了隐喻语言**：这是最重要的前置条件。仅当用户在对话中自然而然地用隐喻表达（"像""如同""被""陷入"等隐喻性的结构）时，隐喻协作才能真正有效。强行引入隐喻会显得生硬与虚假。

2. **隐喻指向核心困境或关键模式**：不是所有用户的隐喻都值得深化。当用户的隐喻触及其行为困难的根本观念（如"我像一只没有翅膀的鸟"指向了"无力感"的根源）时，隐喻协作可以带来深度改变。相反，若隐喻仅是表面的修辞，协作可能过度。

3. **用户处于"State Before Strategy"之后**：用户需要处于相对稳定、有认知资源的状态。高压力、疲劳或高情绪激动状态下，用户难以进行内省式的隐喻探索。

4. **用户显示出内省能力与开放心态**：隐喻协作依赖于用户愿意进行自我对话。如果用户倾向于**外部归因**（"都是别人的错"）或**行动导向但拒绝反思**，隐喻协作效率会大打折扣。

### 3.2 场景适用性

**特别有效的场景：**

- 用户重复陷入同一类困境，且已经反复尝试"常规解决方案"（如节食、运动计划）但屡屡失败。隐喻协作可以帮助识别这些困境背后的*心理结构*，而非仅止于行为层的建议。
- 用户对自己的行为充满矛盾或内疚感（如"我知道这样做不对，但还是做了"）。隐喻协作可以帮助用户理解这种矛盾不是"意志力缺陷"，而是多个系统（理性目标 vs. 情绪需求 vs. 身体状态）的冲突映射。
- 用户处于转折点或身份重组阶段（如从"反复失败者"向"可靠的自己"转变）。隐喻可以作为心理过渡的容器。

**低效或不适用的场景：**

- 用户处于精神卫生危机（自伤、自杀想法等）。此时需要直接的、对接临床资源的干预，隐喻协作会延误必要的医疗行为。
- 用户明确拒绝反思或内省（如"我不想深入分析，我就是想要结果"）。强行进行隐喻协作会被视为教练在逃避提供实际建议。
- 用户的隐喻表达源于语言习惯或文化风格，而非真实的心理困境。此时"深化"隐喻会显得虚伪。

## 4. 不适用场景与禁忌

### 4.1 绝对禁忌

1. **精神卫生合并症**：当用户表现出重度抑郁症、双相障碍、精神分裂症谱系症状或活跃的成瘾行为时，隐喻协作不是一级干预。这些情况需要医学评估与可能的药物治疗，隐喻协作最多只能作为辅助。

2. **急性创伤或高压力状态**：创伤性事件的直接余波期间，来访者的神经系统处于过度警戒状态，此时进行深层心理工作（包括隐喻协作）会加重创伤症状。需要先进行"生理调节"（见 State Before Strategy 原则）。

3. **认知功能障碍**：如果用户存在显著的认知障碍（如老年痴呆、脑损伤引起的认知衰退、严重注意障碍等），隐喻协作所要求的内省与自我观照能力无法满足。

4. **明确的拒绝与不信任**：如果用户明确表示"我不想谈论感受""我不相信这种心理学的东西"，强行推进隐喻协作会损害信任关系。教练应尊重用户的自主权。

### 4.2 相对禁忌与使用过度的风险

1. **内省疲劳**：隐喻协作所需的自我观照是认知密集的。如果用户在同一个会话或相邻会话中已经进行了多轮高强度的自我反思，再次启动隐喻协作会导致"反思疲劳"，反而降低效果。一个原则是：*每个会话周期内最多一次隐喻协作*。

2. **"沉溺于隐喻"的陷阱**：教练与用户可能陷入对隐喻的无止境深化，而失去**实际行动**的目标。隐喻协作的目的是洞察与可能性的打开，而非"完美理解"。当隐喻的深化不再带来新的行动可能性时，应该停止。

3. **隐喻的"虚假安慰"**：有些用户会利用隐喻协作作为逃避行动的借口。例如，用户说"我现在明白了自己像一个'困顿的圆'"，但在后续几周内没有任何行动改变。此时教练需要识别隐喻理解与实际改变之间的断裂，并直面这一点，而非继续围绕隐喻进行对话。

4. **文化不敏感性**：强行将西方隐喻框架（如"容器""障碍"）应用到文化背景不同的用户身上，或忽视用户可能具有的传统哲学倾向，会导致隐喻协作显得生硬或不尊重。教练需要留意文化信号，灵活切换西方与东方路径。

## 5. 对 Voliti 的适用性分析

### 5.1 AI 对话可实现部分 / A2UI 承载部分 / 人类教练才能做的部分

**AI 对话可实现：**

- 隐喻识别与镜射：Coach Agent 可以通过自然语言处理识别用户隐喻（通过标记隐喻性结构如"像""被""陷入"），并以保真方式镜射回去，这是 Clean Language 的第一步。当前 GPT-4 级别的语言模型在这一点上已经表现可靠。

- 中立提问的生成：一系列 Clean Language 式的中立问题（"那个 X 像什么？""当你说 Y 时，会发生什么？"）可以通过提示工程设计为 Coach Agent 的工作流。关键是确保问题保持"中立性"（不暗示答案或价值判断）。

- 东方路径的文本启蒙：禅宗公案式的悖论提问或"取象比类"的自然象征引入，可以被编码为 Agent 的备选提问库。

**A2UI 承载的部分：**

- `ProtocolPromptComponent` 作为开场原语：隐喻协作的启动应采用统一的 `ProtocolPromptComponent`，其中 `observation` 字段明确陈述"用户使用了隐喻 X"，`question` 字段给出第一个镜射或开放性提问。示例：
  - observation: "你说最近像'被淹没'了，喘不了气。"
  - question: "那个被淹没的感觉，现在还在吗？"

- 视觉与节奏的"干预形态"标记：由于隐喻协作涉及多轮对话与深化，前端应通过 `metadata.surface="intervention"` 与 `metadata.intervention_kind="metaphor-collaboration"` 来标识这一形态，从而采用专门的视觉外壳（更多留白、铺垫、仪式化的动效，强调"进入一个对话空间"的感受）。

**人类教练才能做的部分：**

- **真实陪伴与即时调整**：隐喻协作的深层功能不仅是"提问"，而是在用户脆弱的内省时刻提供**真实的人类在场**。AI 无法模拟人类的真正同理心、直觉调整与对时刻细微信号的感应。当用户的声调突然沉默、呼吸改变或措辞转变时，人类教练可以立即感知并据此调整节奏，而 AI 的感知与调整会滞后数秒到数分钟。

- **伦理边界的把守**：隐喻协作可能触及用户的深层创伤、性别认同、宗教信念等极其敏感的领域。人类教练有能力在第一时间识别"我们越过了这个人的伦理边界"，并恰当地停止、转向或转介。AI 在这方面的能力受限。

- **文化与个人背景的实时整合**：虽然 Coach Agent 可以内置"检测用户文化倾向"的信号，但真正地与用户的整个生命故事、代际创伤、族群传统进行对话，需要人类教练的深度理解与灵活响应。

### 5.2 与 S-PDCA 五阶段的接入点

隐喻协作在 S-PDCA 循环中主要作用于 **State（调频）** 与 **Check（模式识别）** 两个阶段：

**State（调频）阶段：**

虽然隐喻协作不是一级的"调频微干预"（那是 ProtocolPrompt 在压力评分驱动下的触发），但当用户在"调频对话"中自发使用了隐喻，Coach 可以在不大幅拉长会话的前提下进行轻量级镜射与一轮提问。例如：

用户说"今天工作太累，我像一块被挤干的海绵"。Coach 可以回应："被挤干的海绵。那个感受，是想停下来，还是想重新吸水？"——这一轮隐喻协作既服务于"理解当前状态"，也在无形中帮助用户从"被动的疲劳受害者"视角转向"有选择权的主体"视角。

**Check（模式识别）阶段：**

当 Coach 在检查用户的长期行为模式时，隐喻协作成为了深化模式理解的工具。假设 Coach 识别到一个模式："用户在每次加班后都会因压力暴食，然后内疚三天。"在 Check 阶段，Coach 可以邀请用户用隐喻来表达这个循环：

Coach: "你注意到加班→暴食→内疚这个循环了吗？用什么隐喻来描述它？"
用户: "像一个齿轮，咔嗒咔嗒，我控制不了。"
Coach: "那个齿轮……是你在推动它，还是它在推动你？"

通过这一轮隐喻协作，用户可能意识到"齿轮"（循环）与"我"（主体）在他的心理表象中其实不是同一个东西，这打开了"我可能改变这个循环"的可能性，进而为后续的 Act（干预调整）阶段铺垫。

**不强烈接入的阶段：**

- **Plan（意图对齐）**：这一阶段主要是目标制定与计划构建，强调"具体性"与"可执行性"。隐喻协作倾向于打开空间与多义性，与 Plan 阶段的"收束与澄清"目标有所冲突。隐喻协作可以在 Plan 前进行（帮助用户理清真实诉求），但不应在 Plan 中进行。

- **Do（行为记录）**：这是行为执行与数据采集的阶段，隐喻协作无直接作用。

- **Act（结构纠偏）**：这一阶段是"基于检查结果进行策略调整"。如果调整涉及行为改变（如环境重塑、应对策略更新），隐喻协作可作为"激发动机"的工具，但不应拖累"行动决策"的速度。

### 5.3 与 Chapter / LifeSign / Process Goal 机制的结合点

**Chapter（章节）与隐喻协作：**

一个 Chapter 代表用户的一个行为改变目标（如"稳定饮食节奏"）。当用户对某个 Chapter 的执行产生了隐喻性的感受或困扰时，隐喻协作可以帮助深化对该 Chapter 困难的理解。

示例：用户在"办公室午餐选择"这个 Chapter 的执行中反复失败。用户说"每次我走进餐厅，我的意志力就像被吸入一个黑洞。"通过隐喻协作，用户可能会发现"黑洞"其实代表"我感到被同事的期待包围"或"餐厅的香气激活了我的渴望，而我觉得抗拒很孤独"。这些洞察可以转化为对该 Chapter 的**触发条件与应对策略**的精化。

**LifeSign 与隐喻协作：**

LifeSign 是"关键行为信号"与"风险预案"的集合。当 Coach 与用户一起回顾 LifeSign 的触发机制时，隐喻可以成为"标签化 LifeSign"的手段。例如，若某个 LifeSign 是"工作压力突增后第二天容易暴食"，Coach 与用户可以给这个模式起一个隐喻性的名字（如"压力释放口"或"自我抚慰的仪式"）。这样，当这个信号再次出现时，用户不是"惊慌失措"，而是"认出了一个熟悉的朋友"，进而有更从容的应对。

**Process Goal 与隐喻协作：**

Process Goal 是在特定情景下的"即刻目标"（如"明天午餐时，选择沙拉而非炸鸡"）。隐喻协作不太适用于 Process Goal 的制定阶段（那需要具体性），但可以在"为什么我总是无法执行这个 Process Goal"的反思中发挥作用。

### 5.4 隐喻如何进入 Coach 四分区记忆

Coach 的记忆分为四个区域：**Verified Patterns（已验证模式）** / **Hypotheses（工作假设）** / **Coaching Notes（教练笔记）** / **Claimed vs Revealed（声称 vs 显露）**。隐喻协作产生的洞察需要被系统化地记录：

**Verified Patterns 区：**

当隐喻协作多次揭示同一个根本观点或心理结构时，这就成为了"已验证模式"。例如，如果用户在不同场景中反复使用"被困""漏气""失控"这些隐喻，教练可以总结"用户倾向于将困难内化为'自己的无力'而非'环境与我的不匹配'"。这一观点进入 Verified Patterns 后，可以成为未来干预决策的基础。

**Hypotheses 区：**

隐喻协作产生的新洞察常常先进入"工作假设"。例如，教练基于一次隐喻对话推测"用户对'控制'的渴望可能源于幼年的不安全感"。这是假设，需要在后续对话中验证。

**Coaching Notes 区：**

每次隐喻协作的内容应被逐字记录（或至少核心隐喻与提问记录）。这既是为了对用户负责（用户可以查阅自己的隐喻历程），也是为了教练在后续对话中保持一致性（避免重复问同样的问题，也避免遗忘关键洞察）。

**Claimed vs Revealed 区：**

用户"声称"的自我理解（如"我就是个意志力弱的人"）与在隐喻协作中"显露"的真实困境（如"我其实害怕成功后失去同伴"）之间往往存在落差。这个落差本身就是有价值的数据，应被妥善记录，以便教练在后续工作中谨慎地桥接这两个视角。

### 5.5 东西方路径在 Voliti 对话场景下的选用准则

**信号与启发式决策规则：**

Coach Agent 不应在系统提示词中硬编码"检测用户文化背景→自动选择东方或西方路径"的逻辑，而应在**用户的实时对话信号**中识别倾向：

1. **语言风格信号**：
   - 如果用户频繁使用"像""似乎""也许"等保留性措辞，或使用传统成语/诗句，Coach 可倾向于东方路径。
   - 如果用户使用"我需要""我想要""应该"等直指性措辞，或出现西方心理学术语（如"压力""焦虑"），Coach 可倾向于西方路径。

2. **思维模式信号**：
   - 用户问"为什么我总是这样？"（因果分析）时，更适合西方 CMT 的"映射"思维。
   - 用户问"我现在该怎么办？"（状态接纳）时，可以引入东方"与困难共存"的智慧。

3. **明确的文化或哲学倾向**：
   - 用户明确表示信任中医、禅学或传统哲学时，教练可在获得明确同意的前提下，邀请用户探索"东方路径"的隐喻协作。

**实现方案（后期可选）：**

在 Coach 的六维用户画像中，可以逐步扩展"Identity & Motivation"维度，加入"隐喻与哲学倾向"的评估（如"用户对隐喻的偏好""传统文化的开放度""对不确定性的耐受度"）。一旦这些维度被结构化，Coach Agent 可以在调用隐喻协作 Skill 时，自动选择更匹配该用户的路径，而无需每次都由 Agent 自行判断。
