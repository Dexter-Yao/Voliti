# LLM输出一致性与召回率最大化

## 一句话简介

同一查询在多次运行中返回不同数量结果的根本原因是三层叠加随机性，完全确定性不可实现，目标应从"每次输出相同"转向"稳定的高召回率"。

## Takeaway

1. **完全确定性不可能** — Gemini 推理模型 + Search Grounding 的组合从根本上排除了确定性输出
2. **目标应是"稳定的高召回率"** — 而非"每次输出完全一致"
3. **成本效益排序**：Structured Output（免费）→ Query Decomposition（1.5x）→ USC（2x）→ PRISM（2.5x）
4. **USC 与 Self-Consistency 有根本区别**：后者 majority voting 选单一答案，不适合搜索场景；USC 取并集才是正确策略
5. **评估框架需要量化一致性**：Jaccard Similarity ≥ 0.7 + Coverage Variance < 0.05 作为"stable"标准

---

## 核心分析

### 三层随机性来源

**Layer 1: 推理模型 GPU 非确定性**
- Gemini 推理模型（2.5 Pro/Flash、3 Pro/Flash）本质不确定
- 即使设置 seed + temperature=0 也无法保证确定性（Google GitHub Issue #745 确认）
- 根因：GPU 浮点运算顺序不确定 + thinking 路径分支差异
- Gemini 3 官方建议：保持 temperature=1.0（降低会导致循环行为）

**Layer 2: Search Grounding 动态查询**（最大随机性来源）
- Grounding 模块根据模型推理动态生成搜索查询，不可控
- 搜索索引实时更新：同一查询在不同时间返回不同结果
- 无官方参数控制 grounding 查询生成策略

**Layer 3: 提取模型变异**
- 下游 LLM 用于筛选和报告生成
- Azure OpenAI 同样存在 GPU 非确定性
- 若无 seed 参数设置，变异无法量化

### 方案对比

| 方案 | 成本倍数 | 召回率提升 | 适用场景 | 实现复杂度 |
|------|----------|-----------|---------|-----------|
| Naive Repeat（重复 N 次取并集） | N × | +17%（N=3） | 简单粗暴 | 低 |
| **USC（Universal Self-Consistency）** | ~2× | +15-20% | 搜索场景最优 | 中 |
| Query Decomposition（查询分解） | ~1.5× | +15% | 高效替代重复 | 中 |
| **Structured Output（结构化输出）** | 1× | 变异 -70% | 格式一致性 | 低 |
| PRISM（迭代检索） | ~2.5× | +25% | 深度召回 | 高 |

### USC（Universal Self-Consistency）— Google DeepMind 2023

核心思想：多次采样 → LLM 元决策合并去重（取**并集**而非交集）。

与 Self-Consistency（Wang et al. 2023）的关键区别：后者用 majority voting 选单一答案，适合推理问题；USC 取并集，适合搜索/召回场景——搜索结果天然是集合形态，USC 策略完美匹配。

实现方式：`repeat_iterations=3` + 后处理 LLM 合并。

### Query Decomposition

将一个搜索查询分解为多个语义等价变体：
- 示例："Tesla legal violations" → ["Tesla SEC enforcement actions", "Tesla lawsuit settlements", "Tesla regulatory fines"]
- 优势：比简单重复更高效（1.5x 成本 vs 3x 成本），且增加语义覆盖面
- 若已有 `repeat_iterations` 机制但每次使用相同查询，改为语义变体是低成本改造

### Structured Output（response_schema）

- 强制模型按 JSON schema 输出
- dottxt.ai 研究显示可降低输出变异约 **70%**
- 零额外成本，仅需修改 API 调用参数

### PRISM 迭代检索

- Selector ⇔ Adder 循环（最多 3 轮），每轮基于已有结果查找遗漏
- 召回率提升最大（+25%），但成本高（2.5x）且延迟增加
- 适合对召回率要求极高的场景，需评估成本效益后再决定

### 优化优先级路线图

- **P0**：Structured Output（零成本，立即可做）——输出格式变异降低约 70%
- **P1**：Query Decomposition（低成本）——+15% 召回率，1.5x 成本
- **P2**：USC 聚合（中等成本）——repeat_iterations=3 + LLM 元决策合并，+15-20% 召回率，约 2x 成本
- **P3**：PRISM 迭代（高成本，远期评估）——+25% 召回率，约 2.5x 成本

### 行业基线

DD（尽职调查）行业召回率基线：**83%**（BlackPeak 案例研究，arxiv:2508.16571）
