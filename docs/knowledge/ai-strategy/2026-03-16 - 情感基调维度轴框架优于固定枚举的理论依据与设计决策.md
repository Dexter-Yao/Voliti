# 情感基调维度轴框架优于固定枚举的理论依据与设计决策

## 一句话简介

固定枚举选项（如"精准权威/温暖活力/开放好奇/简洁高效"）无法覆盖全部设计方向，采用两个底层维度 × 六个体验锚点的框架，在覆盖度与用户可理解性之间取得更好平衡。

## Takeaway

1. **维度轴方法优于固定枚举**：固定选项覆盖范围有限，用户常遇到"都不太对"的困境；维度轴方法将无限可能的设计方向映射到有限坐标系，同时通过锚点描述保持用户可理解性。
2. **Phase 0 的核心职责是赋予词汇**：用户通常不知道如何表达设计偏好，也不确定什么样的描述方式是合适的。锚点描述的功能不只是提供选项，更是为用户提供描述自己期望的语言——选项本身就是教育工具。
3. **两个底层维度足够**：Energy（Still↔Dynamic）× Register（Cool↔Warm）可以覆盖绝大多数产品的情感区间，物理感/触觉质感不作为独立维度——它是多个心理学维度的组合反应，通过 Phase 2 探针自然浮现。
4. **优先级排序不应在 Phase 0 完成**：若用户选了两个方向，不追加主次确认问题——Phase 2 Round 1 通过用户对视觉探针的反应，自然完成优先级判断，比抽象问卷更可靠。

## 问题根源：固定枚举的结构性局限

设计系统构建工具中的情感基调问题（emotional register），通常以固定选项呈现（例如 4 个互斥选项）。这种结构存在两个根本问题：

- **覆盖盲区**：固定项只能覆盖象限中心，无法表达象限之间的渐变区域，也无法覆盖每个象限内部不同的体验层次。
- **强制归类**：用户被迫从不完全贴切的选项中选一个，导致 Phase 2 探针的起点偏移——探针标签基于错误的锚点生成，后续工作在错误的方向上迭代。

## Phase 0 的启发性职责：替用户建立表达词汇

设计系统 Foundation 阶段面对的用户，大多处于三重认知困境：

1. 不知道如何表达自己对设计的喜好（直觉在，语言不在）
2. 不知道设计领域用什么词汇来描述这些偏好
3. 不确定自己的直觉是否"说得通"，是否是合理的设计方向

因此，Phase 0 的情感基调问题不只是信息收集——它本身是一个**词汇赋予工具**（vocabulary scaffold）。六个锚点的描述不是提供候选答案，而是为用户提供一套可以激活他们直觉、并给他们语言去描述的参照系。用户读完描述后的第一反应（"这个很像"/"这个完全不是我想要的"）比他们主动描述的语言携带更多信息。

这个原则也解释了为什么锚点描述必须写成**体验感受**而非设计规则：用户可以判断自己是否想要某种体验，但无法在未见到设计之前判断某种配色或字号是否合适。

## 理论依据：四个研究框架的收敛

**Nielsen Norman Group 四维框架**：Formal↔Casual / Serious↔Funny / Respectful↔Irreverent / Enthusiastic↔Matter-of-fact。最实用的商业框架，强调"品牌声音"（brand voice）由多个独立维度构成，而非单一类型标签。

**Atlassian Voicify**：维度轴 + 滑块定位。商业实践中最成熟的实现方式，证明维度轴对内部设计团队的可操作性。

**Russell's Circumplex Model**：Valence（愉悦度）× Arousal（激活度）。情感心理学的底层坐标系，证明情感状态可以用两个正交维度完整描述。

**Aaker's Brand Personality**：Sincerity / Excitement / Competence / Sophistication / Ruggedness 五维度。说明品牌性格的多个可量化维度不相互排斥，某个品牌可以在多个维度上同时有强表达。

四个框架的共同指向：**情感与品牌性格是多维坐标系，而非离散分类**。固定枚举是对坐标系的粗糙近似，在用途收窄时够用，但在设计系统构建场景中不够。

## 新框架：两个底层维度 × 六个体验锚点

### 底层维度（AI 内部使用）

| 维度 | 两极 |
|---|---|
| **Energy（能量）** | Still/Composed ↔ Dynamic/Kinetic |
| **Register（人际距离）** | Cool/Professional ↔ Warm/Personal |

选择这两个维度的理由：Energy 轴对应 Russell Circumplex 的 Arousal 轴；Register 轴对应设计语境中的人际距离感，与 Valence 轴相关但更适合品牌性格描述。两轴正交，覆盖主流产品设计方向的全域。

### 六个体验锚点

锚点的作用是**将抽象维度坐标翻译为用户可感知的体验描述**，解决"我在坐标系哪里"这个问题对非设计师用户不可理解的困难。

| 锚点 | Energy | Register | Phase 2 探针标签倾向 |
|---|---|---|---|
| Precise Instrument | Still | Cool | Weighty · Precise · Deliberate |
| Institutional Authority | Still | Cool | Formal · Authoritative · Systematic |
| Warm Utility | Still | Warm | Considered · Quiet · Approachable |
| Attentive Companion | Dynamic | Warm | Responsive · Playful · Aware |
| Kinetic System | Dynamic | Cool | Fast · Platform · Decisive |
| Crafted Texture | Either | Warm | Layered · Material · Intentional |

**同象限的两个锚点**：Precise Instrument 与 Institutional Authority 同处 Still+Cool，靠"物质感 vs 系统感"在体验层区分——前者让用户感到"在使用一件被认真制造的实物"，后者让用户感到"在使用一套清晰可信的系统"。用户无需理解维度坐标即可感知差异。

## 关键交互设计决策

**多选而非单选（1–2 项）**：允许用户选 2 项反映两个方向的融合，这比强制单选更接近真实设计方向。融合方向的优先级排序延迟到 Phase 2 完成，不在 Phase 0 追问。

**开放输入（Other）的处理规则**：AI 将用户描述与六个锚点做语义比对，识别最近似的 1–2 个锚点，保留用户原始描述在 foundation.md 中，基于近似锚点标签生成 Phase 2 探针。若描述明显偏离所有锚点，AI 主动追问澄清。

**物理感/触觉不作为 Phase 0 独立问题**：用户在 Phase 0 难以将物理质感与情感方向解耦表达。Precise Instrument 选择后，Phase 2 探针标签自然包含 `Weighty · Tactile` 类词汇，物理感通过视觉探针反应自然确认，而不是在概念问卷阶段被要求预先声明。
