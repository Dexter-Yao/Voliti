# 减脂行为调查 V2 · 设计背景文档

## 一、定位与目标

### 1.1 本轮问卷的定位

第一轮问卷（85 人有效样本）已验证了一系列核心假设。第二轮面向新的公开群体投放，目标不是重复验证，而是**填补盲点、推进决策**。

### 1.2 已验证事实（不再重复）

| 已验证事实 | V1 数据来源 | 置信度 |
|-----------|------------|--------|
| 核心动机：健康 + 外形不满意 | Q3 Borda 排序（85 人） | 高 |
| 首要失控触发：情绪/压力 > 社交 > 出差 | Q5 矩阵 + 3 场访谈 | 高 |
| 79% 偏好私密或有限范围减脂 | Q8 单选 + 访谈一致 | 高 |
| "知行分离"普遍存在（92% 经历过） | Q6 单选 | 高 |
| 四象限分层有效（起步/反复/轻度/重度脱轨） | Q1×Q2 交叉 | 高 |
| 失控后 31% 螺旋型、69% 恢复型 | Q7 单选 | 高 |
| 花费分布：85% 花过钱，多数 < 2000 元 | Q11 单选 | 高 |
| 现有工具"机械"、不动态调整 | 3 场访谈 | 中 |
| 私教主要提供"锚点"价值而非饮食专业度 | 3 场访谈 | 中 |
| 计划在执行环节断裂，非计划本身的问题 | 3 场访谈 | 中 |

## 二、设计原则

### 2.1 Mom Test 原则

所有题目基于用户**过去的真实经历和行为**，不问假设性偏好。唯一例外是 Q12（产品形态排序）和 Q13（推送频率偏好），因为这两题涉及用户对已熟悉的 UX 模式的偏好表达，不涉及对不存在产品的想象。

- ✗ "如果有一位 AI 教练帮你管理减脂，你最希望它……"
- ✓ "您是否认真按照某个人的建议执行过减脂方案？"

### 2.2 认知负荷控制

- 排序题：1 道（仅 4 项排序，用于产品形态偏好）
- 矩阵题：0 道
- 开放题：2 道（选填，面向高意愿用户）
- 多选题限选 2-3 项
- 每个选项以**【关键词】**开头，便于快速扫描

### 2.3 MECE 选项设计

每题选项遵循互斥穷尽原则。使用与 V1 一致的"【关键词】详细描述"格式。

### 2.4 六维框架对齐

问卷变量与 Coach 记忆体系六维框架的映射：

| 六维框架维度 | 对应问卷题号 | 采集深度 |
|------------|------------|---------|
| 维度一：选择空间与摩擦结构 | Q7（饮食方式）、Q9（居住情况）| 结构化 |
| 维度二：身份认同与驱动力 | Q4（归因模式）、Q5（可教练性）、Q19（MBTI）| 结构化+间接推断 |
| 维度三：日常饮食与生活习惯 | Q7（饮食方式）、Q6（计划行为）| 结构化 |
| 维度四：生活节奏与周期模式 | Q10（节奏干扰因素）| 结构化 |
| 维度五：失控场景与触发模式 | V1 已验证；Q20 开放题深挖 | 定性补充 |
| 维度六：工具经验与信任阈值 | Q8+Q8b（AI 信任）、Q14（花费）| 结构化 |

## 三、逐题设计理据

### 板块一：您的减脂经历（Q1-Q5）

#### Q1. 当前状态 [单选]

**驱动决策：** 基本分层 + 非目标人群筛除。

沿用 V1 Q1。D/E 跳至结尾。

#### Q2. 时长与次数 [单选]

**驱动决策：** 与 Q1 交叉构建四象限。

V1 的 F 选项"3年以上，1-5次"产生歧义（6 人需逐个判断归属）。修正为：
- E: 3 年以上，1-2 次（归属"少次"）
- F: 3 年以上，3 次以上（归属"多次"）

频次分界线统一为 1-2 = 少次、3+ = 多次。

#### Q3. 最佳效果 [单选]

**驱动决策：** 区分本质不同的产品需求。

V1 完全缺失此维度。三场访谈中：
- Clare：瘦 20 斤后反弹 → 需要**维持机制**
- Tiantian：运动多+不吃晚饭但减不下来 → 需要**方案适配**
- Raven：请私教期间成功 → 需要**持续结构**

Q3 分布直接决定 Coach 优先做"防反弹"还是"找方法"还是"维持节奏"。

| 选项 | 用户需求类型 | Coach 策略优先级 |
|------|------------|---------------|
| A: 成功维持 | 维持型 | 稳定机制 |
| B: 有效但反弹 | 维持机制 | 防反弹干预 |
| C: 效果有限 | 方案优化 | 个性化方案调整 |
| D: 坚持但无效 | 方案适配 | 方法匹配+体质适配 |
| E: 未能坚持 | 启动与坚持 | 门槛降低+持续参与 |

#### Q4. 失败归因 [单选]

**驱动决策：** Coach 干预语言和策略设计。

基于归因理论（Weiner），每个选项映射一种心智模型和对应的干预策略：

| 选项 | 归因维度 | Coach 策略 |
|------|---------|-----------|
| A: 方案难执行 | 外部、可变 | Fogg B=MAP 降低行为门槛 |
| B: 节奏被打乱 | 外部、不可控 | 环境预警+方案弹性 |
| C: 看不到效果 | 结果依赖 | 进度可见化+阶段成就感 |
| D: 一次就崩 | 内部、固定倾向 | **防螺旋**（Voliti 核心差异点） |
| E: 慢慢松懈 | 渐进耗竭 | Coach 主动触达+持续参与 |

#### Q5. 外部指导经历与期望 [单选]

**驱动决策：** 用户的"可教练性" + 未被满足的专业指导需求。

Mom Test 式题目：基于过去真实行为。新增 D 选项"想请但没条件"区分**行为**与**期望**——有人没请私教不是不想，而是受限于资金或渠道。如果 D 占比高，直接验证"AI 教练作为专业指导的平价替代"这一价值主张。

| 选项 | 信号 | Coach/产品含义 |
|------|------|-------------|
| A: 跟过专业人士 | 高可教练性 | Coach 可更主动、更有方案性 |
| B: 跟过身边人 | 中，信任社会关系 | Coach 需先建立关系 |
| C: 跟过网上方法 | 中，自主筛选型 | Coach 提供选项让用户自选 |
| D: 想请但没条件 | **高意愿低门槛** | **核心价值验证**：AI 教练的定价和可及性优势 |
| E: 自己来 | DIY 导向 | Coach 需极度非指令性 |

**交叉分析价值：**
- Q5(A) × Q3(B/C/D) = "跟过专业人士但仍失败" → Voliti 最高价值目标
- Q5(D) × Q14(花费低) = "想要专业指导但花不起" → 定价策略的直接验证

#### Q6. 减脂计划行为 [单选]

**驱动决策：** Coach onboarding 流程设计——用户需要多少结构化引导。

不同的计划倾向决定 Coach 在 onboarding 阶段应该做什么：

| 选项 | 用户类型 | Coach onboarding 策略 |
|------|---------|-------------------|
| A: 详细计划 | 规划型 | Coach 帮助审视和优化用户自己的计划 |
| B: 大致方向 | 方向型 | Coach 帮助把方向转化为可执行的阶段目标 |
| C: 直接行动 | 行动型 | Coach 在行动中实时引导，不预设方案 |
| D: 搜集信息 | 研究型 | Coach 帮助从信息过载中筛选、聚焦一个方法 |
| E: 等待契机 | 触发型 | Coach 需要帮用户建立内在动机，不依赖外部事件 |

**交叉分析价值：** Q6 × Q4 检验"计划行为"与"失败归因"的一致性——如果"详细计划者"的失败归因是"方案难执行"(Q4-A)，说明问题不在规划能力而在方案灵活性。

### 板块二：您的日常环境（Q6-Q9）

#### Q6. 饮食获取方式 [多选，最多 2 项]

**驱动决策：** Coach 记忆架构"选择空间"维度。

Tiantian 核心洞察："你根本不看我身边我可以点到什么外卖。" Coach 能给出多具体的饮食建议，取决于它对用户食物获取方式的了解。

限选 2 项：大多数人有一到两种主要方式，选太多稀释信号。

#### Q8. AI 聊天工具使用与信任 [单选]

**驱动决策：** Coach 产品的 AI 接受度基线。

选项为渐进式深度，基于真实行为：
- A（没用过）→ AI 教育成本高
- B（只聊工作）→ 场景迁移有门槛
- C（聊过私事但不信）→ 信任是核心瓶颈
- D（聊过且有用）→ 高接受度
- E（经常依赖）→ 最易转化

**交叉分析：** Q8 × Q4 识别"AI-ready + 高需求"的金三角用户。

#### Q8b. AI 信任因素 [多选，最多 2 项，条件触发]

**驱动决策：** Coach 人格设计 + 记忆体系优先级。

仅对 Q8 选 C/D/E 的用户显示——有实际 AI 交互经验的人才能回答。

每个选项直接映射一个 Coach 设计维度：

| 选项 | Coach 设计维度 |
|------|-------------|
| A: 个性化 | onboarding 深度 + 记忆个性化 |
| B: 专业性 | prompt 中的专业措辞和学术引用 |
| C: 记忆力 | 记忆可见性机制（知识库核心原则） |
| D: 态度 | 人格风格（empathetic vs clinical） |
| E: 可执行 | 行动计划的具体度和可行性 |
| F: 不确定 | 兜底 |

如果 A（个性化）+ C（记忆力）占主导，说明六维记忆架构是产品的核心差异化。如果 B（专业性）占主导，说明需要优先建立营养学/运动科学的专业背书。

#### Q9. 居住情况 [单选]

**驱动决策：** 环境摩擦结构；Coach 饮食干预空间。

居住情况是食物环境和社交动态的强代理变量：
- 独居 → 深夜点外卖风险高（Raven），但食物储备可控
- 与家人 → 回家后不会点外卖（Raven 访谈），但社交进食不可控
- 合住 → 食物获取受限（如宿舍取外卖高摩擦）
- 经常不固定 → 出差型，节奏最不稳定（Clare）

#### Q9. 生活节奏干扰因素 [多选，最多 3 项]

**驱动决策：** Coach 需要应对的核心场景类型。

V1 的"工作节奏"题过于粗糙（4 个选项无法区分具体干扰模式）。本题改为直接问过去 3 个月的**真实干扰事件**：

| 选项 | Coach 干预映射 |
|------|-------------|
| A: 出差/异地 | 出差模式：提前预案+临时调整 |
| B: 晚间应酬 | 社交进食策略：餐前预案+事后不审判 |
| C: 作息不规律 | 灵活时间框架，不依赖固定时刻 |
| D: 周末放纵 | 周末专属计划，不照搬工作日节奏 |
| E: 独自进食 | 情绪性进食风险预警 |
| F: 都不常见 | 稳定节奏，Coach 可做更结构化的方案 |

"过去 3 个月至少发生过 2 次"锚定到真实频率，避免假设性回答。

### 板块三：产品与触达偏好（Q10-Q14）

#### Q10. 主动使用健康管理工具的习惯 [单选]

**驱动决策：** 平台架构（Web vs iOS）。

基于真实行为："你现在有没有主动打开过健康类 App？"

| Q10 结果 | Q13 通知偏好 | 平台含义 |
|---------|-------------|---------|
| 主动使用 | 不需推送 | Web 版可行 |
| 主动使用 | 期待推送 | iOS 更优 |
| 不主动 | 期待推送 | **iOS 必须** |
| 不主动 | 排斥推送 | 需重新思考产品形态 |

#### Q11. 推送通知频率偏好 [单选]

**驱动决策：** 平台决策 + Coach 主动性节奏设计。

选项覆盖从高频到零推送的完整谱系，每个选项同时包含**频率**和**态度**：

| 选项 | 频率 | Coach 设计 |
|------|------|-----------|
| A: 每天 3 次以上 | 高频 | 全天候陪伴模式 |
| B: 每天 1-2 次 | 适度 | 日常 briefing + 晚间复盘 |
| C: 一周几次 | 低频 | 关键节点触达 |
| D: 关键时刻 | 事件驱动 | JITAI 式干预 |
| E: 不需要 | 零 | 纯被动响应 |
| F: 没概念 | 未知 | 兜底 |

如果 A+B（每天至少 1 次）占主导，说明用户期待 Coach 主动出现在日常中——iOS 推送是核心产品能力。如果 D+E 占主导，Web 版可行。

#### Q12. 产品形态偏好 [排序题]

**驱动决策：** 开发优先级排序。

4 项排序 + 1 个开放选项，认知负荷低。排序能看到第二偏好的分布——如果"App 第一 + 小程序第二"占绝对主导，说明移动端是共识。

#### Q13. 信息来源 [多选，最多 2 项]

**驱动决策（双重）：**
1. 竞争替代方案识别
2. GTM 渠道策略

#### Q14. 过去一年花费 [单选]

**驱动决策：** 用户对减脂的实际经济投入。

沿用 V1 Q11 的经过验证的选项。真实花费行为比假设性 willingness to pay 可靠得多。

### 板块四：基本信息（Q15-Q19）

#### Q15. 性别 [单选]

沿用 V1。

#### Q16. 年龄段 [单选]

沿用 V1。

#### Q17. 所在城市级别 [单选]

不同城市级别的饮食环境、健康餐覆盖度、付费能力有结构性差异。

#### Q19. 体重状况自评 [单选]

**驱动决策：** Coach 方案强度设计 + 用户分群。

大基数和小基数用户的需求本质不同：

| 选项 | 核心需求 | Coach 策略差异 |
|------|---------|-------------|
| A: 大基数（15 斤+） | 健康驱动，生活方式调整 | 更长周期规划，阶段性里程碑，可能涉及医学边界 |
| B: 中等基数（5-15 斤） | 健康+外形，实际可达 | 最主流群体，标准 Coach 方案 |
| C: 小基数（5 斤内/塑形） | 线条/体态，精细管理 | 体重秤不是主要指标，需要体态/围度等替代指标 |
| D: 不确定 | 缺乏自我评估 | onboarding 需先建立身体认知 |
| E: 不愿透露 | 敏感话题 | 尊重隐私 |

选项用"斤"作为锚点而非 BMI——"斤"是中国用户最直觉的单位，BMI 需要计算且很多人不知道自己的 BMI。"大基数""小基数"是减脂社区通用术语，用户一看就知道自己属于哪一类。

**交叉分析价值：**
- Q19 × Q3（效果）：大基数用户是否更容易"有效但反弹"？
- Q19 × Q5（可教练性）：大基数用户是否更倾向于寻求专业指导？
- Q19 × Q14（花费）：不同体重群体的经济投入差异

#### Q20. MBTI 性格类型 [单选]

**驱动决策：** 用户群体的性格特征画像 + 聚类分析维度。

MBTI 在中国年轻用户群体中普及度高，大部分人知道自己的类型。本题目的不是做性格测试，而是作为聚类变量观察性格特征与减脂行为模式的关联：

- **J vs P 维度**（计划型 vs 灵活型）可能与 Q6（计划行为）强相关
- **T vs F 维度**（逻辑型 vs 情感型）可能影响 Coach 沟通风格偏好
- **E vs I 维度**（外向 vs 内向）可能与隐私偏好（V1 已验证 79% 偏好私密）和社交进食模式相关

"不知道"和"不愿透露"选项确保不给用户压力。

## 四、分析计划

### 4.1 聚类变量矩阵

| 变量 | 类型 | 框架维度 |
|------|------|---------|
| Q1 | 有序分类 | 行为状态 |
| Q2 | 有序分类 | 经历深度 |
| Q3 | 有序分类 | 效果历史 |
| Q4 | 名义分类 | 心智模型（归因） |
| Q5 | 名义分类 | 可教练性 + 未满足需求 |
| Q6 | 名义分类 | 计划倾向 |
| Q7 | 多选→二进制 | 环境（饮食获取） |
| Q8 | 有序分类 | AI 接受度 |
| Q9 | 名义分类 | 环境（居住） |
| Q10 | 多选→二进制 | 节奏干扰模式 |
| Q14 | 有序分类 | 付费能力 |
| Q17 | 名义分类 | 城市级别 |
| Q19 | 名义分类 | 体重基数（方案强度） |
| Q20 | 名义分类 | MBTI（J/P 维度尤其重要） |

目标：识别 2-4 个自然聚类，回答"谁是高价值目标用户，他们的共同特征是什么"。

预期聚类假设：
- **Cluster A "AI-Ready 反复行动者"**：Q2=多次, Q3=反弹, Q5=A/D(跟过专业人士或想请), Q8=D/E → 首要目标
- **Cluster B "想要指导但没条件"**：Q5=D, Q14=低花费, Q8=C/D → 定价验证群体
- **Cluster C "节奏失控者"**：Q10=出差+应酬, Q4=节奏被打乱 → 灵活性优先的产品
- **Cluster D "DIY 研究型"**：Q5=E, Q6=D(搜集信息), Q8=A/B → 需要教育成本高
- **Cluster E "低意愿者"**：Q1=C, Q3=E, Q14=未花钱 → 非目标

### 4.2 关键交叉分析
