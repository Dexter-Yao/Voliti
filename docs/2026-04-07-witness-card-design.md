# Witness Card 见证系统设计

**日期**：2026-04-07
**状态**：设计定稿，待实现
**前置依赖**：`docs/01_Product_Foundation.md` 4.1 节（教练干预体系 + 见证系统）

---

## 一、问题陈述

### 1.1 现有系统的评估

对当前签名体验图片系统进行了完整评估：使用 5 种干预模板 + 3 个扩展场景，通过 Azure OpenAI gpt-image-1.5 生成 8 张样本图片。评估结论：

**技术层面表现尚可**：文字渲染准确（中英文均可）、情绪基调基本匹配、Warm/Cool 风格分化有效。

**产品层面全部不合格**：
- 风格不统一——5 种模板各自为政，用户无法形成"这是 Voliti 的"品牌认知
- 意图不清晰——部分图片过于抽象（identity_evolution）、过于晦暗（高风险场景）或缺乏个性化锚点
- 留存价值低——定位为"一次性干预工具"，看完即走，不值得保存和回看
- 与用户场景脱节——使用通用意象而非用户的具体经历

### 1.2 根因

问题不在图片生成技术，而在**产品定位错位**。

当前系统把图片定位为**教练干预工具**（Coach 觉得"文字不够用"时的替代品）。但用户真正需要的是**旅程中的里程碑纪念品**（达成成就后可以永久收藏的东西）。

这两个定位在触发时机、个性化锚点、情绪目标、留存价值上完全不同。

## 二、设计决策

### D1. 干预与见证分离

**决策**：图片只用于见证（witnessing），不用于干预（intervention）。

**理由**：如果在用户心智中图片既是干预工具又是纪念品，会引起认知混淆。一张图片不能同时是"帮你改变行为的工具"和"记录你成就的纪念品"。

**执行**：
- 四类干预方法（未来自我对话、场景预演、隐喻协作、认知重构）全部通过 Coach 文字对话 + A2UI 结构化交互实现
- 图片生成仅在里程碑时刻触发，产出 Witness Card

### D2. 见证定位为 witnessing，不是 rewarding

**决策**：Coach 生成 Witness Card 的语气是"我记录了你的这个时刻"，而非"给你一个奖品"。

**理由**：基于自我决定理论（SDT），信息性反馈支持内在动机，控制性奖励侵蚀内在动机。将图片定位为"奖励"会触发过度辩护效应——对已内在动机化的行为引入外部奖励，反而降低动力。

### D3. 统一视觉体系，Coach 选择色温

**决策**：所有 Witness Card 共享同一品牌框架和视觉语言，Coach 在体系内选择色温方向。

**理由**：统一框架让用户一眼认出"这是 Voliti 的"，同时色温变化（暖色=成长时刻、冷色=力量时刻）保留情感表达空间。

### D4. 文图分离

**决策**：图片不嵌入文字，文字在卡片框架的独立区域呈现。

**理由**：
- 图片内嵌文字经常模糊、排版失控（评估中发现的问题）
- 文字独立呈现时可读性更高，意义更清晰
- 用户回看时能快速获取语义（不需要凑近看图片里的小字）

### D5. FanOutPanel 呈送，支持收下/拒绝

**决策**：通过现有 A2UI 的 FanOutPanel interrupt 流程呈送 Witness Card。

**理由**：发送 Witness Card 的过程本身就是教练引导——Coach 告诉用户"你做到了什么"，用户选择是否收下。拒绝不影响 Coach 对用户的理解。

## 三、Witness Card 结构

### 3.1 卡片布局

```
┌─────────────────────────────┐
│  [Voliti 品牌标识]     [日期] │
│                              │
│  ┌────────────────────────┐  │
│  │                        │  │
│  │    AI 生成的场景图片    │  │
│  │    （无文字叠加）       │  │
│  │                        │  │
│  └────────────────────────┘  │
│                              │
│  "连续21天，你每天早上都选择  │
│   了先喝水再看手机。"        │
│                              │
│  ── 你的第一个 Chapter 完成  │
│                              │
│            [用户名]          │
└─────────────────────────────┘
```

### 3.2 卡片元素

| 元素 | 来源 | 说明 |
|------|------|------|
| 品牌标识 | 固定 | Voliti logo / wordmark |
| 日期 | Coach 判断 | 里程碑发生的日期 |
| 场景图片 | AI 生成 | 基于用户具体经历，无文字叠加 |
| 叙事文字 | Coach 撰写 | 个性化的 Coach 语气文字，引用用户原话 |
| 成就标题 | Coach 撰写 | 简短的里程碑描述 |
| 用户名 | Store | 来自用户 profile |

### 3.3 统一视觉体系

**不变元素（品牌一致性）**：
- 插画/水墨/水彩风格，非写实渲染
- 纸张质感（handmade paper texture）
- 大面积留白（≥40%）
- 避免真人面孔（剪影、抽象形态）
- 去饱和、柔和色调
- 无游戏化元素

**可变元素（Coach 决定）**：

色温轴，根据里程碑的情感基调：

- **暖色调**（成长、温柔、被看见）：羊皮纸背景 #F5F1EB + 深黑曜石 #1A1A1A
- **冷色调**（突破、力量、掌控）：暖黑背景 #2A2520 + 低饱和冷蓝 #8AACB8

## 四、触发逻辑

### 4.1 原则

由 Coach 自主判断，不硬编码指标。Coach 基于对话上下文和 ledger 数据综合判断"这个时刻是否值得见证"。

### 4.2 信号分类

**显性成就**：用户主动告知的成功。
- "我昨晚商务晚宴选了气泡水"
- "我这周每天都自己做了早餐"
- "我跟同事说了不想喝酒，他们居然都接受了"

**隐性成就**：Coach 通过 ledger 数据发现的、用户自己未必意识到的进步。
- 连续 N 天 Check-in（用户可能没注意到自己的一致性）
- 某类 LifeSign 预案连续成功多次
- 行为模式的正向转变（如晚间暴食频率从每周 3 次降至 0 次）

隐性成就的见证价值更高——用户没意识到自己做到了什么，Coach 主动为他生成 Witness Card 说"你知道吗，这周你每天都......"。这个"被看见"的体验是教练关系的核心价值。

**旅程节点**：
- Onboarding 完成（启程仪式，首张 Witness Card）
- Chapter 完成（约 21 天）
- LifeSign 累计成功里程碑

### 4.3 干预成功 → 见证的闭环

```
Coach 检测到风险/机会
  → Coach 用文字 + A2UI 进行干预（场景预演、认知重构等）
    → 用户成功应对
      → Coach 识别这是里程碑时刻
        → 生成 Witness Card
          → 用户收下 → 存入收藏
```

干预和见证在时间和角色上完全分开：干预发生在风险时刻（前置），见证发生在成功之后（后置）。

### 4.4 频率约束

每 Chapter（约 21 天）约 3-5 张。里程碑的稀缺性是其情感重量的来源。如果每次对话都生成 Witness Card，用户会迅速脱敏。

## 五、完整教练干预体系

Witness Card 是更大的教练干预体系的一部分。完整体系分为 SESSION 内和 SESSION 间两层：

```
SESSION 内（聊天窗口）
├── 对话式教练 ─── Coach 文字对话（核心干预手段）
│   ├── 状态评估 ─── A2UI（slider, select）
│   ├── 场景预演 ─── A2UI + 文字引导
│   ├── 认知重构 ─── A2UI + 文字引导
│   ├── 隐喻协作 ─── A2UI + 文字引导
│   └── 未来自我 ─── A2UI + 文字引导
│
│ ↓ 当用户达成里程碑时
│
└── Witness Card ─── FanOutPanel（品牌框架 + 图片 + 文字）
    → 用户收下 → 存入收藏
    → 用户拒绝 → Coach 尊重，继续对话

SESSION 间（主动触达，P2）
├── 前瞻提醒 ─── Forward Marker 触发
├── Check-in ─── 时间节奏触发
├── 模式洞察 ─── JourneyAnalysis 触发
└── 回看引导 ─── 低动力期触发（引导用户回看 Witness Cards）
```

**闭环**：Push 把用户拉回来 → 对话中 Coach 进行干预 → 干预成功产生成就 → 成就触发 Witness Card → Card 收藏积累为旅程证据 → 低动力期 Push 引导回看 → 用户获得力量再次进入对话。

## 六、收藏与回看

### 6.1 存储

已接受的 Witness Card 存入 LangGraph Store（命名空间 `("voliti", "user", "interventions")`），包含完整图片数据、叙事文字、成就标题、日期和色温方向。

### 6.2 回看机制

在 MIRROR Tab 中展示用户的 Witness Card 收藏。按时间排列，用户可以随时浏览自己的旅程证据。

在低动力期，Coach 可主动引导用户回看——利用怀旧（nostalgia）驱动的自我连续感（self-continuity）重新激发目标追求行为。Sedikides (2023) 的研究表明，自我连续感直接预测目标追求行为。

### 6.3 删除

用户可随时删除已收藏的 Witness Card，删除不影响 Coach 对用户的理解。

## 七、学术支撑

| 理论 | 来源 | 在 Witness Card 中的应用 |
|------|------|------------------------|
| 自我决定理论（SDT） | Ryan et al., 2023 | 见证（信息性反馈）优于奖励（控制性奖励） |
| 自我连续感 | Sedikides, Hong & Wildschut, 2023 | 回看历史 Witness Card 增强过去-现在-未来自我的连接 |
| 怀旧与目标追求 | Sedikides & Wildschut, 2025 | 怀旧促进趋近动机和成长导向 |
| 双重编码理论 | Paivio, 1971 | 图片 + 文字同时激活两条神经通路，记忆留存更强 |
| 前后对比图有害 | Hales et al., 2024 (RCT, N=404) | 绝对不用并排前后对比图（升高自我客体化） |
| 图像优越效应 | 多项研究 | 柔和对比、低信息密度的图片留存效果最佳 |

## 九、开放问题

1. **卡片框架的精确视觉规格**：品牌标识位置、字体、间距、圆角等精确 design tokens 待定。需要与 `DESIGN.md` 和 design-tokens.json 协调。
2. **图片风格的最终锚定**：统一视觉体系的方向已确定（水墨/水彩 + 色温轴），但缺少视觉参考锚点。需要生成新一轮样本图片验证调整后的 prompt 策略。
3. **生成时间优化**：当前 gpt-image-1.5 生成时间 ~57 秒，用户体验不佳。需验证 Azure 端是否有优化空间，或考虑 quality="medium" 两阶段策略。
