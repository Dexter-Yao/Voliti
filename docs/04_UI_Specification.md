# 线框图设计规范

## 一、底部标签栏（三页签通用）

采用 Signal Tower 风格：纯文字、等宽体、大写英文。

```
┌─────────────────────────────────────┐
│                                     │
│           （页面内容区）              │
│                                     │
├─────────────────────────────────────┤
│   COACH        MAP       JOURNAL    │
│   ─────                             │
└─────────────────────────────────────┘
```

- 高度：44px
- 字体：等宽体（SF Mono / IBM Plex Mono），12px，字间距 2px
- 当前页签：黑曜石色，下方短下划线
- 非当前页签：40% 灰色
- 三等分布局，无分隔线
- 背景：与页面内容同色（羊皮纸色），顶部一条 1px 细线分隔

### 对话区域

- Coach 消息：左对齐，无气泡，衬线体（Noto Serif SC），深黑曜石色
- 用户消息：右对齐，无气泡，无衬线体（Inter / SF Pro），深黑曜石色
- 不使用气泡背景色——消息仅通过对齐方向区分
- 消息间距：16px
- 时间戳：仅在间隔 > 30 分钟时显示，等宽体，12px，灰色 40%，居中
- 背景：纯羊皮纸色（#F5F1EB）
- 无头像

### 输入栏

- 高度：48px，底部固定
- 左侧：「＋」按钮（圆形，黑曜石描边），点击展开拍照 / 相册选项
- 中间：单行文字输入框，无边框，placeholder「消息输入框...」
- 右侧：语音按钮（🎤图标，长按录音）+ 发送按钮（→ 箭头，有文字时变黑）
- 整体以细线与对话区域分隔

## 三、关于「偏航」——为什么这个词不属于本产品

### 1. 「阻抗被视为信息，而非错误」（01_Product_Foundation.md）

「偏航」隐含二元判断——在正确路径上 vs 偏离了正确路径。但产品的核心假设是：用户吃多了、跳过运动、在压力下选择了安慰食物——这些都是**有价值的信息**，反映了当时的状态、环境和需求。标记为「偏航」就是在说「你做错了」，这与「阻抗 = 信息」直接矛盾。

### 2. Identity First，而非 Performance First

「偏航」是性能语言——它属于导弹制导系统或航线控制。它把人简化为「是否在执行计划」的二元状态。但产品关注的是身份演化：「你正在成为什么样的人」。一个人在应酬晚餐中吃多了，不代表他的身份偏航了——他可能同时是一个重视社交关系的人。

### 3. Map Over Metrics

「偏航」本质上是一个度量概念——距离目标路径的偏差值。它把体验缩减为数字。但设计哲学是「Map Over Metrics」——用户在一张地图上，不是在一条数轴上。地图上没有「偏航」，只有「你在这里」。

### 替代框架：所有行走都是路径的一部分

- 不说「你偏航了」→ 说「这一天路径更宽」或「经过了不同的地形」
- 不说「你回到了正轨」→ 说「节奏恢复了」或「这几天的路径很稳」
- 不说「风险」→ 说「这里地形复杂」或「这种情境需要更多注意力」
- 不用红色标记某天 → 用路径宽度、色彩饱和度、地形变化来表达差异

- 逆时间序（最近在上）
- 日期头：衬线体，16px，带星期
- 时间戳：等宽体，12px，灰色
- 事件类型：无衬线体，14px，黑曜石色
- 数据摘要：等宽体，12px，灰色
- 标签：方括号包裹，行内排列
- 条目间细线分隔（1px，黑曜石 10%）

**缺席日处理**：如果用户某天没有任何记录，Journal 中不显示该天日期头——只展示有数据的日子，不制造「你没有记录」的暗示。

**周期标记**：周一和月初的日期头使用稍粗的分隔线（2px，黑曜石 15%），微妙暗示时间周期的起点。

**设计定位**：详见`/docs/02_Design_System.md`第三节"信息层级结构"。Map页不再承载Horizon/Path Layer的直接呈现，转为Coach在关键时刻生成的教练洞察归档。

1. **Chapter 元数据**（顶部）：当前目标、身份宣言、已进行天数

- Coach 文案完整显示（列表中可能被截断）

Map 页不显示"缺席"概念——仅展示 Coach 已生成的卡片。用户没有卡片时显示空状态，不制造"你没有记录"的暗示。

## 六、Coach 页补充规范

以下补充规范与第二节 Coach 页基础设计协同，不替代。

- **确信度标注**：趋势卡片等数据呈现扇出中，标注 Coach 分析的数据基础。如：「Coach 分析（基于 12 天数据）」
- **坏消息传递**：当 Coach 需要指出摄入过高等负面信息时，使用更简洁、更结构化的语言（数据先行），减少情感化表达
- **推理过程展示**：Protocol Prompt Card 展示 Coach 的观察链（「我注意到你的压力分数连续两天 > 7，这通常出现在…之前」），引导用户自己形成洞察，而非直接给指令
